## 1. Why Go Was Built

Go was created at Google in 2007 (released 2009) by Rob Pike, Ken Thompson, and Robert Griesemer — the people who built Unix, Plan 9, and UTF-8. Their problem: C++ compilation times at Google were measured in hours, and existing languages didn't handle modern multi-core CPUs well.

**Go's design goals:**
- Compile in seconds (not minutes)
- Statically typed but feel dynamically typed
- Built-in concurrency primitives for multi-core CPUs
- Garbage-collected but low-latency
- Simple — fewer concepts, one way to do things

The result: a language that is **boring by design**. That's a feature, not a bug. Every Go codebase looks like every other Go codebase. No metaprogramming, no operator overloading, no inheritance hierarchies. Just straightforward, readable code.

## 2. Go's Core Concurrency: Goroutines

A **goroutine** is Go's lightweight concurrent unit. It is NOT an OS thread — it's managed by the Go runtime.

```go
package main

import (
    "fmt"
    "time"
)

func fetchUser(id int) string {
    time.Sleep(100 * time.Millisecond)  // simulates DB query
    return fmt.Sprintf("user-%d", id)
}

func main() {
    // Sequential: 300ms total
    u1 := fetchUser(1)
    u2 := fetchUser(2)
    u3 := fetchUser(3)
    fmt.Println(u1, u2, u3)

    // Concurrent with goroutines: ~100ms total
    ch := make(chan string, 3)  // buffered channel

    go func() { ch <- fetchUser(1) }()
    go func() { ch <- fetchUser(2) }()
    go func() { ch <- fetchUser(3) }()

    results := []string{<-ch, <-ch, <-ch}
    fmt.Println(results)
}
```

**Goroutine cost vs. OS thread:**
```
OS Thread:   ~1-8 MB stack, ~1µs to create, OS-scheduled
Goroutine:   ~2-8 KB stack (grows dynamically), ~0.3µs to create, Go-scheduled

10,000 goroutines: ~80 MB RAM (feasible)
10,000 OS threads: ~10-80 GB RAM (not feasible)

→ You can have millions of goroutines. You cannot have millions of OS threads.
```

## 3. Channels: Communication Between Goroutines

Go's philosophy: *"Don't communicate by sharing memory; share memory by communicating."*

Channels are typed conduits for passing data between goroutines.

```go
// Unbuffered channel: sender blocks until receiver is ready
ch := make(chan int)
go func() { ch <- 42 }()   // blocks until main goroutine receives
val := <-ch                 // blocks until goroutine sends
fmt.Println(val)            // 42

// Buffered channel: sender only blocks when buffer is full
ch := make(chan int, 5)
ch <- 1   // doesn't block (buffer has space)
ch <- 2   // doesn't block
val := <-ch  // 1

// Select: wait on multiple channels (like a switch for channels)
select {
case msg1 := <-ch1:
    fmt.Println("from ch1:", msg1)
case msg2 := <-ch2:
    fmt.Println("from ch2:", msg2)
case <-time.After(5 * time.Second):
    fmt.Println("timeout")
}
```

## 4. Go for Backend Services

```go
package main

import (
    "encoding/json"
    "log"
    "net/http"
    "strconv"

    "github.com/go-chi/chi/v5"
    "github.com/jackc/pgx/v5/pgxpool"
)

type User struct {
    ID    int    `json:"id"`
    Name  string `json:"name"`
    Email string `json:"email"`
}

func main() {
    pool, err := pgxpool.New(ctx, os.Getenv("DATABASE_URL"))
    if err != nil { log.Fatal(err) }
    defer pool.Close()

    r := chi.NewRouter()

    r.Get("/users/{id}", func(w http.ResponseWriter, r *http.Request) {
        id, err := strconv.Atoi(chi.URLParam(r, "id"))
        if err != nil {
            http.Error(w, "Invalid ID", http.StatusBadRequest)
            return
        }

        var user User
        err = pool.QueryRow(r.Context(),
            "SELECT id, name, email FROM users WHERE id=$1", id,
        ).Scan(&user.ID, &user.Name, &user.Email)

        if err != nil {
            http.Error(w, "User not found", http.StatusNotFound)
            return
        }

        w.Header().Set("Content-Type", "application/json")
        json.NewEncoder(w).Encode(user)
    })

    log.Printf("Listening on :8080")
    http.ListenAndServe(":8080", r)
}
```

**Key Go patterns for backend:**

```go
// Error handling: explicit, no exceptions
result, err := db.Query("SELECT ...")
if err != nil {
    return fmt.Errorf("querying users: %w", err)  // wrap error with context
}
defer result.Close()

// Context: deadline propagation (critically important)
ctx, cancel := context.WithTimeout(r.Context(), 5*time.Second)
defer cancel()  // always defer cancel to prevent context leak

rows, err := pool.Query(ctx, "SELECT ...")  // query respects the timeout

// Defer: cleanup (runs when function exits, even on error)
conn, err := pool.Acquire(ctx)
if err != nil { return err }
defer conn.Release()  // always released, no "finally" needed
```

## 5. The Go Standard Library

Go's standard library is comprehensive and production-ready — you often don't need third-party packages:

```
net/http        — HTTP client and server (production-worthy alone)
encoding/json   — JSON marshal/unmarshal
database/sql    — database interface (use with drivers like pgx)
context         — deadline and cancellation propagation
sync            — Mutex, WaitGroup, Once, Map
crypto/*        — TLS, AES, SHA, HMAC
time            — time, duration, ticker, timer
os / io / bufio — system and I/O operations
testing         — built-in test framework
log / log/slog  — logging (slog: structured logging, Go 1.21+)
```

## 6. When to Choose Go

```
GO EXCELS AT:
  ✓ High-throughput APIs (network-bound: 50,000-100,000+ req/s per instance)
  ✓ Services with high concurrency requirements
  ✓ Systems tools (Docker, Kubernetes, Terraform are all written in Go)
  ✓ Microservices where deployment footprint matters (single binary, no runtime)
  ✓ When predictable latency matters (GC pauses are low and predictable)
  ✓ Teams that want strong typing without Java's verbosity

GO IS WORSE AT:
  ✗ Rapid prototyping (more verbose than Python)
  ✗ Data science / ML (Python ecosystem is unmatched)
  ✗ CRUD apps where development speed > performance
  ✗ GUI applications
  ✗ Generic algorithms (generics added in 1.18 but ecosystem is still maturing)

COMPANIES USING GO AT SCALE:
  Docker, Kubernetes, Terraform, GitHub CLI, InfluxDB,
  Cloudflare (edge workers written in Go), Uber, Twitch
```

## 7. Senior Engineer Perspective

**What a junior gets wrong:**

Not using `context.Context` for every database query and HTTP call. In Go, contexts propagate deadlines and cancellation signals. If a user's HTTP request times out, `ctx.Done()` fires — and any in-progress database queries with that context are automatically cancelled. Without context: the DB query runs even after the client disconnected. Always accept and propagate contexts.

**Go's biggest advantage that isn't obvious:**

Single binary deployment. A Go binary includes the runtime, all dependencies, and the entire application. No Python version mismatch, no `node_modules`, no JVM installation. `scp binary server:/usr/local/bin/myapp && ssh server myapp` — that's your entire deployment for many services. In containers, this means FROM scratch images of 10-20 MB instead of 500 MB Python images.
