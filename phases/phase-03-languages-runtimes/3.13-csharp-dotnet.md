## 1. C# and .NET in Context

C# is Microsoft's primary programming language. .NET is its runtime and framework ecosystem. Together they dominate:
- Windows enterprise software
- Microsoft Azure cloud services
- Game development (Unity uses C#)
- Financial services and trading systems in Europe
- Healthcare systems

.NET was Windows-only until .NET Core (2016). Since then, it runs on Linux, macOS, and Windows — and ASP.NET Core is one of the highest-performance web frameworks in any language.

## 2. The CLR (Common Language Runtime)

```
.NET LANGUAGES → CLR (JIT) → Machine Code

Languages: C#, F#, VB.NET, PowerShell all compile to CIL (Common Intermediate Language)
CLR handles: JIT compilation, GC, type safety, security

Similar to JVM but:
  - RyuJIT (the JIT compiler) produces highly optimized code
  - Modern .NET GC: server GC mode for multi-core, with very low pause times
  - Native AOT (Ahead-Of-Time): compile to native binary (like Rust/Go)
    Fast startup, smaller deployment, no JIT warmup needed
```

## 3. ASP.NET Core: Performance Champion

ASP.NET Core consistently ranks among the top-performing web frameworks in TechEmpower benchmarks — often beating Go and Node.js.

```csharp
// Minimal API style (ASP.NET Core 6+) — concise, fast
var builder = WebApplication.CreateBuilder(args);
builder.Services.AddDbContext<AppDbContext>();
var app = builder.Build();

app.MapGet("/users/{id}", async (int id, AppDbContext db) =>
    await db.Users.FindAsync(id) is User user
        ? Results.Ok(user)
        : Results.NotFound());

app.MapPost("/users", async (CreateUserRequest req, AppDbContext db) => {
    var user = new User { Name = req.Name, Email = req.Email };
    db.Users.Add(user);
    await db.SaveChangesAsync();
    return Results.Created($"/users/{user.Id}", user);
});

app.Run();

// Full controller style (traditional, still widely used)
[ApiController]
[Route("api/[controller]")]
public class UsersController : ControllerBase
{
    private readonly IUserService _userService;

    public UsersController(IUserService userService)
        => _userService = userService;

    [HttpGet("{id}")]
    public async Task<ActionResult<UserDto>> GetUser(int id)
    {
        var user = await _userService.GetByIdAsync(id);
        return user is null ? NotFound() : Ok(user);
    }
}
```

C# consistently leads other languages in adopting modern features:

```csharp

    Circle c when c.Radius > 10  => $"Large circle r={c.Radius}",
```

## 5. When to Choose C#/.NET

```
C#/.NET IS EXCELLENT FOR:
  ✓ Microsoft Azure environments (deep integration)
  ✓ Windows enterprise software
  ✓ Teams with existing .NET expertise
  ✓ High-performance APIs (benchmarks rival Go/Rust)
  ✓ Game development backends (Unity servers)
  ✓ Financial systems in Europe (strong .NET tradition)
  ✓ When you need Windows-specific APIs

C#/.NET OUTSIDE ITS STRENGTHS:
  ✗ Linux-native infrastructure tooling (Go dominates)
  ✗ Data science (Python dominates)
  ✗ Startups (Python/Node/Go have more momentum)
  ✗ When the team has no .NET experience
```

## 6. Senior Engineer Perspective

**.NET's underrated strength:**

ASP.NET Core's performance is genuinely world-class. In TechEmpower benchmarks, it outperforms Express.js by 3-5x on raw throughput, approaches Go's performance, and has a richer type system than either. If your team has C# experience and you need high-performance backend, .NET is a legitimate top-tier choice.

**The Windows trap:**

Early .NET (.NET Framework) was Windows-only. .NET Core/.NET 5+ is fully cross-platform. But the ecosystem still has Windows-specific libraries and tooling assumptions. Verify any library you use is cross-platform before committing.
