## 1. What Are They? (Theory)

LangChain and LlamaIndex are frameworks that provide abstractions for common LLM application patterns — RAG pipelines, agent tools, multi-model chains, document loaders, and more. They reduce the boilerplate of building LLM-powered applications.

**The trade-off statement upfront:**

These frameworks are genuinely useful for prototyping but often introduce more complexity than they remove in production. Understanding both their value and their costs helps you make the right choice.

---

## 2. LangChain: The Swiss Army Knife

LangChain (Harrison Chase, 2022) provides abstractions for chaining LLM calls, tools, retrievers, and memory into applications.

```python
# LangChain: building a RAG pipeline

from langchain_anthropic import ChatAnthropic
from langchain_openai import OpenAIEmbeddings
from langchain_community.vectorstores import PGVector
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_core.runnables import RunnablePassthrough

# 1. Initialize components
llm = ChatAnthropic(model="claude-sonnet-4-6")
embeddings = OpenAIEmbeddings(model="text-embedding-3-small")

# Connect to existing pgvector database
vectorstore = PGVector(
    connection_string="postgresql://...",
    collection_name="knowledge_base",
    embedding_function=embeddings,
)
retriever = vectorstore.as_retriever(
    search_kwargs={"k": 5, "score_threshold": 0.7}
)

# 2. Define the RAG chain
prompt = ChatPromptTemplate.from_template("""
Answer the question based only on the following context:
{context}

Question: {question}
""")

def format_docs(docs):
    return "\n\n".join(doc.page_content for doc in docs)

# LangChain Expression Language (LCEL): chain components together
rag_chain = (
    {"context": retriever | format_docs, "question": RunnablePassthrough()}
    | prompt
    | llm
    | StrOutputParser()
)

# Use the chain
answer = rag_chain.invoke("What is our refund policy?")
```

### LangChain's LCEL (LangChain Expression Language)

```python
# LCEL chains using | operator (like Unix pipes)
from langchain_core.runnables import RunnableLambda

# SEQUENTIAL CHAIN: translate then summarize
translate = ChatPromptTemplate.from_template("Translate to Spanish: {text}") | llm | StrOutputParser()
summarize = ChatPromptTemplate.from_template("Summarize in one sentence: {text}") | llm | StrOutputParser()

pipeline = translate | (lambda text: {"text": text}) | summarize

# PARALLEL CHAIN: run multiple tasks simultaneously
from langchain_core.runnables import RunnableParallel

parallel = RunnableParallel(
    summary=summarize,
    translation=translate,
)
results = parallel.invoke({"text": "Your input text here"})
print(results["summary"])
print(results["translation"])

# STREAMING (streaming output token by token)
for chunk in rag_chain.stream("What is the refund policy?"):
    print(chunk, end="", flush=True)
```

## 3. LlamaIndex: Document-First Framework

LlamaIndex (Jerry Liu, 2022) is more focused on document retrieval and indexing — building knowledge bases from diverse document types:

```python
from llama_index.core import VectorStoreIndex, SimpleDirectoryReader, Settings
from llama_index.llms.anthropic import Anthropic
from llama_index.embeddings.openai import OpenAIEmbedding
from llama_index.vector_stores.postgres import PGVectorStore

# Configure global settings
Settings.llm = Anthropic(model="claude-sonnet-4-6")
Settings.embed_model = OpenAIEmbedding(model="text-embedding-3-small")

# Load documents from various sources
documents = SimpleDirectoryReader(
    input_dir="./docs",
    required_exts=[".pdf", ".txt", ".md"],
).load_data()

# Build index with pgvector storage
vector_store = PGVectorStore.from_params(
    database="mydb",
    host="localhost",
    table_name="llama_embeddings",
    embed_dim=1536,
)

index = VectorStoreIndex.from_documents(
    documents,
    storage_context=storage_context,
    show_progress=True,
)

# Query
query_engine = index.as_query_engine(similarity_top_k=5)
response = query_engine.query("What are the key product features?")
print(response.response)
print("\nSources:")
for node in response.source_nodes:
    print(f"  - {node.metadata.get('file_name', 'unknown')}: {node.score:.2f}")
```

## 4. When to Use These Frameworks vs. Building from Scratch

```
USE LANGCHAIN/LLAMAINDEX WHEN:
  ✓ Prototyping quickly (2-day hackathon → they save hours)
  ✓ Using many document formats (PDF, Word, HTML, etc.) — their loaders handle parsing
  ✓ Experimenting with different RAG strategies
  ✓ Small to medium scale where performance isn't critical
  ✓ You want to compare multiple LLM providers quickly

CONSIDER BUILDING FROM SCRATCH WHEN:
  ✗ Performance is critical (framework abstractions add overhead)
  ✗ You need fine-grained control over prompts and retrieval
  ✗ The framework's abstractions don't fit your use case
  ✗ Team has strong Python skills and can build it in a day
  ✗ Debugging framework internals is harder than debugging your own code
  ✗ Long-term maintenance (framework upgrades can break your code)

THE HONEST ASSESSMENT (2024):
  LangChain v0.1 → v0.2 → v0.3: significant API changes between versions.
  Production teams have been burned by needing to rewrite large portions
  when upgrading. The abstractions leak — you end up debugging framework internals.
  
  Many production teams start with LangChain for prototyping,
  then rewrite core components in plain Python once they understand the requirements.
  This is a perfectly valid approach: use frameworks to learn, build custom for prod.
```

## 5. Senior Engineer Perspective

**The "demo → production" gap:**

LangChain and LlamaIndex excel at getting a demo working in a day. The problem comes when you try to make it production-ready: handle errors properly, add retry logic, tune the retrieval strategy, add observability. At that point, the framework's abstractions often become obstacles.

The pragmatic pattern many teams use:
1. Build the prototype with LangChain (fast iteration, understand the problem)
2. Profile where time is spent, identify the actual requirements
3. Rebuild the core pipeline in plain Python + the underlying SDKs directly
4. Keep LangChain for utility functions (document loaders, text splitters)

**LangSmith for observability:**

LangChain's companion product LangSmith provides excellent observability for LLM pipelines — tracing each chain step, viewing prompts and responses, evaluating output quality. Even if you don't use LangChain in production, LangSmith is worth evaluating for its debugging and evaluation capabilities.
