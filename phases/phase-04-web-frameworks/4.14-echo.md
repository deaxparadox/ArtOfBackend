## Echo (4.14)

Echo (2015, Vishal Rana) is a fast, minimalist Go web framework with a slightly cleaner API than Gin:

```go
import "github.com/labstack/echo/v4"

e := echo.New()
e.Use(middleware.Logger())
e.Use(middleware.Recover())

e.GET("/users/:id", func(c echo.Context) error {
    id, _ := strconv.Atoi(c.Param("id"))
    user, err := db.GetUser(c.Request().Context(), id)
    if err != nil {
        return echo.NewHTTPError(http.StatusNotFound, "User not found")
    }
    return c.JSON(http.StatusOK, user)
})

e.POST("/users", func(c echo.Context) error {
    var req CreateUserRequest
    if err := c.Bind(&req); err != nil {
        return echo.NewHTTPError(http.StatusBadRequest, err.Error())
    }
    if err := c.Validate(req); err != nil {
        return echo.NewHTTPError(http.StatusUnprocessableEntity, err.Error())
    }
    user, _ := db.CreateUser(c.Request().Context(), req)
    return c.JSON(http.StatusCreated, user)
})

e.Logger.Fatal(e.Start(":8080"))
```

**Echo vs. Gin:** Similar feature set. Echo's error handling is slightly cleaner (`return echo.NewHTTPError(...)`). Gin's validation (`binding` struct tags) is slightly more convenient. Choose either — both are excellent.

## Fiber (4.15)

Fiber (2020) is a Go framework inspired by Express, built on **fasthttp** (a faster HTTP implementation than `net/http`):

```go
import "github.com/gofiber/fiber/v2"

app := fiber.New()

app.Get("/users/:id", func(c *fiber.Ctx) error {
    id, _ := c.ParamsInt("id")
    user, err := db.GetUser(c.Context(), id)
    if err != nil {
        return c.Status(404).JSON(fiber.Map{"error": "Not found"})
    }
    return c.JSON(user)
})

app.Listen(":8080")
```

**Caveat:** Fiber's fasthttp context is not `net/http`-compatible. Many Go libraries (AWS SDK, database drivers) expect `net/http`. This can cause integration problems. **Prefer Gin/Echo/Chi unless you specifically need Fiber's performance numbers.**

## Chi (4.16)

Chi (2015, Peter Kieltyka) is the most idiomatic Go router — it uses `net/http`-compatible handlers throughout:

```go
import "github.com/go-chi/chi/v5"

r := chi.NewRouter()
r.Use(middleware.Logger)
r.Use(middleware.Recoverer)

r.Route("/users", func(r chi.Router) {
    r.Use(AuthMiddleware)      // applied to all /users routes
    r.Get("/", listUsers)
    r.Post("/", createUser)
    r.Route("/{userID}", func(r chi.Router) {
        r.Get("/", getUser)
        r.Put("/", updateUser)
        r.Delete("/", deleteUser)
    })
})

http.ListenAndServe(":8080", r)
```

**Chi's advantage:** Pure `http.Handler` compatibility. Any middleware written for `net/http` works with Chi. If you value idiomatic Go and ecosystem compatibility over a custom framework API, Chi is the right choice.

## Go Framework Decision Guide

```
Choose net/http (stdlib) if:
  - < 10 routes, Go 1.22+
  - Maximum simplicity, no dependencies
  
Choose Gin if:
  - Most popular, largest ecosystem
  - Team from Node.js/Express
  
Choose Echo if:
  - Slightly cleaner error handling than Gin
  - Similar to Gin, either works

Choose Chi if:
  - Most idiomatic Go
  - Need stdlib-compatible handlers
  - Using existing net/http middleware

Avoid Fiber unless:
  - Proven performance bottleneck requires fasthttp
  - No need for standard library compatibility
```
