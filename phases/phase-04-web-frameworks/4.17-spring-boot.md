## 1. What Is Spring Boot?

Spring Boot (2014, Pivotal) is an opinionated configuration of the Spring Framework that removes the XML configuration nightmare of classic Spring. It provides auto-configuration, embedded servers (Tomcat/Jetty), production-ready actuator endpoints, and starter dependencies.

**Spring Boot's position:** The dominant Java web framework. Every large company with Java uses it or Spring. Understanding Spring Boot is understanding enterprise Java.

## 2. Core Spring Boot

```java
// Spring Boot 3.x with Kotlin (the modern way)
@RestController
@RequestMapping("/api/v1/users")
@RequiredArgsConstructor
class UserController(private val userService: UserService) {

    @GetMapping("/{id}")
    fun getUser(@PathVariable id: Long): ResponseEntity<UserDto> =
        userService.findById(id)
            .map { ResponseEntity.ok(it) }
            .orElse(ResponseEntity.notFound().build())

    @PostMapping
    @ResponseStatus(HttpStatus.CREATED)
    fun createUser(@Valid @RequestBody request: CreateUserRequest): UserDto =
        userService.create(request)

    @DeleteMapping("/{id}")
    @ResponseStatus(HttpStatus.NO_CONTENT)
    fun deleteUser(@PathVariable id: Long) = userService.delete(id)
}

// Validation with jakarta.validation
data class CreateUserRequest(
    @field:Email
    @field:NotBlank
    val email: String,

    @field:NotBlank
    @field:Size(min = 1, max = 100)
    val name: String,
)

// Service layer
@Service
@Transactional
class UserService(private val userRepository: UserRepository) {

    fun findById(id: Long): Optional<UserDto> =
        userRepository.findById(id).map { it.toDto() }

    fun create(request: CreateUserRequest): UserDto {
        if (userRepository.existsByEmail(request.email)) {
            throw UserAlreadyExistsException("Email already registered")
        }
        return userRepository.save(User(email = request.email, name = request.name)).toDto()
    }
}

// Repository (Spring Data JPA)
interface UserRepository : JpaRepository<User, Long> {
    fun existsByEmail(email: String): Boolean
    fun findByEmail(email: String): Optional<User>
}
```

## 3. Spring Boot's Key Features

**Dependency Injection (IoC Container):**

Spring's core: define beans (components), let Spring wire them together. No manual `new` calls for services, repositories, or utilities.

```java
@Component          // generic Spring-managed bean
@Service            // business logic bean (semantic)
@Repository         // data access bean (enables DB exception translation)
@Controller         // web layer bean
@Configuration      // configuration class (defines other beans)
```

**Auto-configuration:**

Add `spring-boot-starter-data-jpa` to `pom.xml` → Spring Boot automatically:
- Configures Hibernate/JPA
- Sets up a DataSource
- Creates entity manager
- Enables `@Transactional`

No XML configuration needed. Convention: `spring.datasource.url` in `application.yml`.

**Spring Security:**

```java
@Configuration
@EnableWebSecurity
class SecurityConfig {

    @Bean
    fun securityFilterChain(http: HttpSecurity): SecurityFilterChain {
        return http
            .csrf { it.disable() }
            .authorizeHttpRequests {
                it.requestMatchers("/api/v1/public/**").permitAll()
                it.requestMatchers("/api/v1/admin/**").hasRole("ADMIN")
                it.anyRequest().authenticated()
            }
            .oauth2ResourceServer { it.jwt { } }
            .build()
    }
}
```

## 4. Spring Boot Production Configuration

```yaml
# application.yml
spring:
  datasource:
    url: ${DATABASE_URL}
    hikari:
      maximum-pool-size: 20
      minimum-idle: 5
      connection-timeout: 30000
  jpa:
    hibernate:
      ddl-auto: validate        # never auto-create in production
    show-sql: false
    properties:
      hibernate.default_batch_fetch_size: 100

server:
  port: 8080
  tomcat:
    threads:
      max: 200
      min-spare: 10

management:
  endpoints:
    web:
      exposure:
        include: health,info,metrics,prometheus
  endpoint:
    health:
      show-details: always
```

**Spring Actuator** provides production-ready endpoints:
- `GET /actuator/health` → health check (Kubernetes readiness/liveness)
- `GET /actuator/metrics` → JVM metrics, request metrics
- `GET /actuator/prometheus` → Prometheus-compatible metrics

## 5. Spring Boot's Weaknesses

```
STARTUP TIME: 10-30 seconds (JVM warmup + Spring context initialization)
  → Bad for: Lambda, scale-to-zero, frequent deploys
  → Solutions: GraalVM native (fast startup), Quarkus, Micronaut

MEMORY: 200-500MB per instance
  → Higher than Go or Node.js for equivalent functionality
  → Justified in enterprise environments with cost-of-developer > cost-of-RAM

COMPLEXITY: Spring has a steep learning curve
  → Autoconfiguration can be magical and hard to debug
  → "Why isn't this bean being created?" is a common Stack Overflow question
  → Spring Boot DevTools + Spring Boot Actuator help significantly
```

## 6. Senior Engineer Perspective

**The Spring Boot reality:** For large enterprises with Java expertise and complex domain models, Spring Boot is genuinely excellent. The ecosystem (Spring Security, Spring Data, Spring Cloud, Spring Batch) is unmatched. Transactional management, JPA integration, and security configuration are all production-tested at extreme scale.

**The JPA N+1 problem is Spring's #1 performance issue:**

```java
// WRONG: N+1 queries
for (Order order : orderRepository.findAll()) {
    System.out.println(order.getUser().getEmail());  // separate query per order!
}

// RIGHT: JOIN FETCH
@Query("SELECT o FROM Order o JOIN FETCH o.user WHERE o.status = :status")
List<Order> findByStatusWithUser(@Param("status") String status);

// OR: use @EntityGraph
@EntityGraph(attributePaths = {"user"})
List<Order> findByStatus(String status);
```

Always use Hibernate statistics in development to see actual query counts, or enable `spring.jpa.show-sql=true`.
