## 1. What Is Gin?

Gin (2014, Manu Martínez-Almeida) is the most popular Go web framework by GitHub stars and npm/pkg.go.dev downloads. It provides an Express-like routing API on top of `net/http`, with middleware support, JSON binding, and validation.

**Why Gin is popular:** Familiar API for Express developers, fast httprouter underneath, large ecosystem of Gin middleware.

```go
package main

import (
    "net/http"
    "github.com/gin-gonic/gin"
)

type CreateUserRequest struct {
    Email string `json:"email" binding:"required,email"`
    Name  string `json:"name"  binding:"required,min=1,max=100"`
}

func main() {
    r := gin.New()

    // Middleware
    r.Use(gin.Logger())
    r.Use(gin.Recovery())  // panic → 500, not crash

    // Route groups
    api := r.Group("/api/v1")
    api.Use(AuthMiddleware())

    // Routes
    api.GET("/users/:id", func(c *gin.Context) {
        id, err := strconv.Atoi(c.Param("id"))
        if err != nil {
            c.JSON(http.StatusBadRequest, gin.H{"error": "Invalid ID"})
            return
        }

        user, err := db.GetUser(c.Request.Context(), id)
        if err != nil {
            c.JSON(http.StatusNotFound, gin.H{"error": "Not found"})
            return
        }
        c.JSON(http.StatusOK, user)
    })

    api.POST("/users", func(c *gin.Context) {
        var req CreateUserRequest
        if err := c.ShouldBindJSON(&req); err != nil {
            c.JSON(http.StatusUnprocessableEntity, gin.H{"error": err.Error()})
            return
        }
        user, err := db.CreateUser(c.Request.Context(), req)
        if err != nil {
            c.JSON(http.StatusInternalServerError, gin.H{"error": "Internal error"})
            return
        }
        c.JSON(http.StatusCreated, user)
    })

    r.Run(":8080")
}

func AuthMiddleware() gin.HandlerFunc {
    return func(c *gin.Context) {
        token := c.GetHeader("Authorization")
        if token == "" {
            c.AbortWithStatusJSON(http.StatusUnauthorized, gin.H{"error": "Unauthorized"})
            return
        }
        // validate token...
        c.Set("user_id", userID)
        c.Next()  // continue to handler
    }
}
```

## 2. Gin vs. Chi vs. Echo

```
GIN:     Most popular, Express-like, validation built in
         Best for: teams from Node.js, want familiar API
         
CHI:     Idiomatic Go, stdlib-compatible handlers, composable
         Best for: teams who want the most Go-idiomatic approach

ECHO:    Slightly cleaner API than Gin, similar performance
         Best for: teams who want a slightly more opinionated alternative to Gin

All three are significantly more convenient than raw net/http for > 5 routes.
Performance differences between them are negligible for real-world workloads.
```
