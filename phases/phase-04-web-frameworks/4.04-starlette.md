## 1. What Is Starlette?

Starlette (2019, Tom Christie) is a lightweight ASGI framework and toolkit. FastAPI is built directly on top of Starlette — FastAPI adds Pydantic validation and OpenAPI documentation, while Starlette provides the underlying routing, middleware, requests, responses, WebSockets, and background tasks.

**Why backend engineers need to know Starlette:**
- Understanding Starlette = understanding how FastAPI works under the hood
- Middleware in FastAPI is Starlette middleware
- WebSocket handling in FastAPI is Starlette WebSocket handling
- For very minimal, high-performance APIs: Starlette without FastAPI overhead

## 2. Starlette Basics

```python
from starlette.applications import Starlette
from starlette.routing import Route, Mount, WebSocketRoute
from starlette.requests import Request
from starlette.responses import JSONResponse, Response
from starlette.middleware import Middleware
from starlette.middleware.cors import CORSMiddleware
from starlette.websockets import WebSocket

async def homepage(request: Request) -> JSONResponse:
    return JSONResponse({"hello": "world"})

async def user_detail(request: Request) -> JSONResponse:
    user_id = request.path_params["user_id"]
    return JSONResponse({"id": user_id})

async def websocket_endpoint(websocket: WebSocket):
    await websocket.accept()
    while True:
        data = await websocket.receive_text()
        await websocket.send_text(f"Echo: {data}")

app = Starlette(
    debug=False,
    routes=[
        Route("/", homepage),
        Route("/users/{user_id:int}", user_detail),
        WebSocketRoute("/ws", websocket_endpoint),
    ],
    middleware=[
        Middleware(CORSMiddleware, allow_origins=["*"]),
    ],
)
```

## 3. Starlette's Key Components Used by FastAPI

```python
# These are Starlette classes, used directly in FastAPI:

from starlette.requests import Request        # request object
from starlette.responses import (
    JSONResponse,         # {"key": "value"} → application/json
    HTMLResponse,         # HTML string
    RedirectResponse,     # 301/302 redirect
    FileResponse,         # serve a file
    StreamingResponse,    # streaming response body
)
from starlette.background import BackgroundTask, BackgroundTasks
from starlette.middleware.base import BaseHTTPMiddleware
from starlette.testclient import TestClient   # sync test client
from starlette.websockets import WebSocket, WebSocketDisconnect

# StreamingResponse — for large responses or Server-Sent Events
import asyncio

async def event_stream():
    for i in range(100):
        yield f"data: {i}\n\n"
        await asyncio.sleep(0.1)

@app.get("/events")
async def stream_events():
    return StreamingResponse(
        event_stream(),
        media_type="text/event-stream",
        headers={"Cache-Control": "no-cache", "X-Accel-Buffering": "no"},
    )
```

## 4. When to Use Starlette Directly

```
USE STARLETTE DIRECTLY (without FastAPI) WHEN:
  ✓ Extreme performance requirements (no Pydantic overhead)
  ✓ Building a framework on top of Starlette
  ✓ Serving static files + websockets + minimal routing
  ✓ Proxy/gateway service with no request body parsing

USE FASTAPI (Starlette + Pydantic) WHEN:
  ✓ Building an API that receives/returns JSON
  ✓ You want automatic documentation
  ✓ You want dependency injection
  ✓ 99% of API use cases

STARLETTE PERFORMANCE:
  Starlette alone: ~140,000 req/s (simple hello world)
  FastAPI:         ~100,000 req/s (Pydantic validation overhead)
  For comparison: Express.js ~40,000 req/s, Django ~15,000 req/s
```

## 5. Senior Engineer Perspective

**Understanding Starlette makes you a better FastAPI developer:**

When a FastAPI middleware behaves unexpectedly, or a WebSocket connection breaks, or a StreamingResponse doesn't flush — the answer is always in Starlette's source code. Reading it (it's very readable) demystifies the "magic" in FastAPI and makes you capable of debugging any framework behavior.
