## 1. What Is Tornado?

Tornado (2009, FriendFeed/Facebook) was the first widely-used async Python web framework — predating asyncio by 3 years. It was groundbreaking: it proved Python could handle tens of thousands of concurrent connections.

Today, Tornado's niche has been absorbed: FastAPI/Starlette replaced it for new async APIs, and Tornado's native async model (`@gen.coroutine`) was superseded by Python's `async/await` syntax. Tornado 6.x uses `asyncio` under the hood but keeps its older API.

**When you encounter Tornado:** Legacy codebases, especially those built 2010-2016 for WebSockets or long-polling.

```python
import tornado.web
import tornado.ioloop

class UserHandler(tornado.web.RequestHandler):
    async def get(self, user_id: int):
        user = await self.application.db.get_user(user_id)
        if not user:
            self.set_status(404)
            self.write({"error": "Not found"})
            return
        self.write({"id": user.id, "email": user.email})

    async def post(self):
        data = tornado.escape.json_decode(self.request.body)
        user = await self.application.db.create_user(data)
        self.set_status(201)
        self.write({"id": user.id})

app = tornado.web.Application([
    (r"/users/([0-9]+)", UserHandler),
])

if __name__ == "__main__":
    app.listen(8888)
    tornado.ioloop.IOLoop.current().start()
```

**Production use today:** Primarily maintaining existing systems. New projects should choose FastAPI/Starlette.
