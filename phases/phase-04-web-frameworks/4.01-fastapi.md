## 1. What Is FastAPI?

FastAPI (2018, Sebastián Ramírez) is a modern, fast, async-first Python web framework for building APIs. It is built on Starlette (ASGI framework) and Pydantic (data validation).

**Why it became the standard for new Python APIs:**
- **Automatic OpenAPI/Swagger docs** — generated from type annotations, always up-to-date
- **Pydantic validation** — request bodies, query params, path params validated automatically
- **Async-first** — built on Starlette/ASGI, handles concurrency efficiently
- **Type safety** — full IDE support, catches bugs at development time
- **Performance** — one of the fastest Python frameworks (rivals Node.js)

FastAPI is now the most popular Python web framework for new API projects, having overtaken Flask and challenging Django's dominance.

## 2. Core Concepts

### Route Definition

```python
from fastapi import FastAPI, HTTPException, Query, Path
from pydantic import BaseModel, EmailStr, Field
from typing import Annotated

app = FastAPI(
    title="My API",
    description="API documentation generated from code",
    version="1.0.0",
)

# Path parameter with validation
@app.get("/users/{user_id}")
async def get_user(
    user_id: Annotated[int, Path(gt=0, description="The user ID")],
):
    user = await db.get_user(user_id)
    if not user:
        raise HTTPException(status_code=404, detail="User not found")
    return user

# Query parameters
@app.get("/users")
async def list_users(
    page: Annotated[int, Query(ge=1, default=1)],
    limit: Annotated[int, Query(ge=1, le=100, default=20)],
    search: str | None = None,
):
    return await db.list_users(page=page, limit=limit, search=search)

# Request body with Pydantic
class CreateUserRequest(BaseModel):
    email: EmailStr
    name: Annotated[str, Field(min_length=1, max_length=100)]
    age: Annotated[int, Field(ge=13, le=120)]

class UserResponse(BaseModel):
    id: int
    email: str
    name: str

    class Config:
        from_attributes = True  # allow creating from SQLAlchemy models

@app.post("/users", response_model=UserResponse, status_code=201)
async def create_user(data: CreateUserRequest):
    user = await db.create_user(data)
    return user
```

### Dependency Injection

FastAPI's DI system is its most powerful feature — and the cleanest DI in any Python framework:

```python
from fastapi import Depends
from sqlalchemy.ext.asyncio import AsyncSession
import redis.asyncio as aioredis

# Database session dependency
async def get_db() -> AsyncSession:
    async with AsyncSessionLocal() as session:
        try:
            yield session
            await session.commit()
        except Exception:
            await session.rollback()
            raise

# Redis dependency
async def get_redis() -> aioredis.Redis:
    redis = aioredis.from_url(settings.redis_url)
    try:
        yield redis
    finally:
        await redis.aclose()

# Auth dependency
async def get_current_user(
    token: str = Depends(oauth2_scheme),
    db: AsyncSession = Depends(get_db),
) -> User:
    payload = verify_jwt(token)
    user = await db.get(User, payload["sub"])
    if not user:
        raise HTTPException(status_code=401, detail="Invalid token")
    return user

# Admin-only dependency (chains on get_current_user)
async def require_admin(current_user: User = Depends(get_current_user)) -> User:
    if not current_user.is_admin:
        raise HTTPException(status_code=403, detail="Admin access required")
    return current_user

# Using dependencies in routes
@app.delete("/users/{user_id}", status_code=204)
async def delete_user(
    user_id: int,
    db: AsyncSession = Depends(get_db),
    admin: User = Depends(require_admin),  # only admins can delete
):
    user = await db.get(User, user_id)
    if not user:
        raise HTTPException(404, "User not found")
    await db.delete(user)
```

### Background Tasks

```python
from fastapi import BackgroundTasks

@app.post("/orders", status_code=201)
async def create_order(
    data: CreateOrderRequest,
    background_tasks: BackgroundTasks,
    db: AsyncSession = Depends(get_db),
    current_user: User = Depends(get_current_user),
):
    order = await db.create_order(data, current_user.id)

    # Schedule email after response is sent — doesn't block
    background_tasks.add_task(
        send_order_confirmation_email,
        email=current_user.email,
        order_id=order.id,
    )

    return order
```

## 3. Middleware

```python
from starlette.middleware.base import BaseHTTPMiddleware
from starlette.middleware.cors import CORSMiddleware
import time, uuid

# CORS middleware
app.add_middleware(
    CORSMiddleware,
    allow_origins=settings.allowed_origins,
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

# Custom request ID + timing middleware
class RequestContextMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request, call_next):
        request_id = request.headers.get("x-request-id", str(uuid.uuid4()))
        start = time.perf_counter()

        # Bind to logging context
        with structlog.contextvars.bound_contextvars(request_id=request_id):
            response = await call_next(request)

        duration_ms = (time.perf_counter() - start) * 1000
        response.headers["x-request-id"] = request_id
        response.headers["x-response-time"] = f"{duration_ms:.2f}ms"
        return response

app.add_middleware(RequestContextMiddleware)
```

## 4. Lifespan Events (Startup/Shutdown)

```python
from contextlib import asynccontextmanager
from sqlalchemy.ext.asyncio import create_async_engine, AsyncSession
from sqlalchemy.orm import sessionmaker

engine = None
AsyncSessionLocal = None

@asynccontextmanager
async def lifespan(app: FastAPI):
    # STARTUP
    global engine, AsyncSessionLocal
    engine = create_async_engine(settings.database_url)
    AsyncSessionLocal = sessionmaker(engine, class_=AsyncSession)
    app.state.redis = aioredis.from_url(settings.redis_url)
    print("Application started")

    yield  # app runs here

    # SHUTDOWN
    await engine.dispose()
    await app.state.redis.aclose()
    print("Application shut down")

app = FastAPI(lifespan=lifespan)
```

## 5. Error Handling

```python
from fastapi import Request
from fastapi.responses import JSONResponse
from fastapi.exceptions import RequestValidationError

# Validation errors (422 responses)
@app.exception_handler(RequestValidationError)
async def validation_exception_handler(request: Request, exc: RequestValidationError):
    return JSONResponse(
        status_code=422,
        content={
            "error": "validation_failed",
            "details": [
                {
                    "field": ".".join(str(loc) for loc in err["loc"][1:]),
                    "message": err["msg"],
                    "type": err["type"],
                }
                for err in exc.errors()
            ],
        },
    )

# Generic unhandled exceptions
@app.exception_handler(Exception)
async def generic_exception_handler(request: Request, exc: Exception):
    request_id = getattr(request.state, "request_id", "unknown")
    logger.exception("Unhandled exception", request_id=request_id)
    return JSONResponse(
        status_code=500,
        content={"error": "internal_error", "request_id": request_id},
    )
```

## 6. Production Setup

```python
# Full production FastAPI application structure

myapp/
├── main.py              # FastAPI app, lifespan, middleware registration
├── config.py            # Pydantic Settings
├── database.py          # SQLAlchemy engine, session factory
├── dependencies.py      # Shared dependencies (db, auth, redis)
├── models/
│   └── user.py          # SQLAlchemy models
├── schemas/
│   └── user.py          # Pydantic request/response models
├── routers/
│   ├── users.py         # /users routes
│   └── orders.py        # /orders routes
├── services/
│   └── user_service.py  # Business logic
└── tests/
    └── test_users.py
```

```python
# main.py
from fastapi import FastAPI
from app.routers import users, orders
from app.middleware import RequestContextMiddleware
from starlette.middleware.cors import CORSMiddleware

app = FastAPI(title="My API", lifespan=lifespan)

# Middleware (order matters — outermost first)
app.add_middleware(RequestContextMiddleware)
app.add_middleware(CORSMiddleware, allow_origins=["*"])

# Routers
app.include_router(users.router, prefix="/api/v1", tags=["users"])
app.include_router(orders.router, prefix="/api/v1", tags=["orders"])
```

```bash
# Run in production
gunicorn main:app \
    -w 4 \
    -k uvicorn.workers.UvicornWorker \
    --bind 0.0.0.0:8000 \
    --timeout 30 \
    --graceful-timeout 30
```

## 7. FastAPI vs. Django vs. Flask

| Factor | FastAPI | Django | Flask |
|---|---|---|---|
| Async | Native | Added (3.1+) | Extension |
| Auto docs | Yes | No | No |
| ORM | None (bring your own) | Built-in | None |
| Admin | No | Yes | Extension |
| Validation | Pydantic (excellent) | Forms (limited) | Extension |
| Learning curve | Low | Medium | Very low |
| Best for | APIs, microservices | Full-stack apps | Simple APIs |
| Performance | High | Medium | Medium |

## 8. Senior Engineer Perspective

**What a junior gets wrong:**

Not using `response_model`. FastAPI will serialize everything returned by a handler. If your handler returns a SQLAlchemy model with 50 fields (including passwords, internal fields), all 50 are serialized. Always define a `response_model` (Pydantic schema) that includes only what clients should see.

**What a senior always adds:**

1. `response_model` on every endpoint — controls what's returned
2. `status_code` on POST (201), DELETE (204) — not just 200 everywhere
3. Exception handlers for both `RequestValidationError` and `Exception`
4. Structured logging middleware with request ID
5. Health check endpoint at `/health` or `/ready`
6. API versioning prefix: `/api/v1/`

**FastAPI's hidden superpower:**

The `Depends()` system composes perfectly. You can chain dependencies arbitrarily deep: `require_admin` depends on `get_current_user` which depends on `get_db`. Each dependency is tested independently, mocked in tests, and reused across hundreds of routes. This is better DI than most other frameworks in any language.
