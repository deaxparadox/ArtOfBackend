## 1. What Is Fastify?

Fastify (2016, Matteo Collina and Tomas Della Vedova) is a Node.js web framework designed for maximum performance and developer experience. Key design decisions: **JSON schema for serialization** (not JSON.stringify), **plugin architecture** (not middleware), and **native TypeScript support**.

**Performance reason:** Fastify uses `fast-json-stringify` — it generates optimized serialization code from your JSON Schema. This is 2-3x faster than `JSON.stringify` for typical API responses. Combined with a fast router and lower overhead, Fastify consistently outperforms Express in benchmarks.

## 2. Fastify Core

```typescript
import Fastify, { FastifyRequest, FastifyReply } from 'fastify';
import { z } from 'zod';

const fastify = Fastify({
    logger: true,  // built-in pino logger
});

// Schema-validated route (TypeScript)
interface UserParams { id: string; }
interface UserResponse { id: number; name: string; email: string; }

fastify.get<{ Params: UserParams; Reply: UserResponse | { error: string } }>(
    '/users/:id',
    {
        schema: {
            params: {
                type: 'object',
                properties: { id: { type: 'string' } },
                required: ['id'],
            },
            response: {
                200: {
                    type: 'object',
                    properties: {
                        id: { type: 'number' },
                        name: { type: 'string' },
                        email: { type: 'string' },
                    },
                },
                404: {
                    type: 'object',
                    properties: { error: { type: 'string' } },
                },
            },
        },
    },
    async (request, reply) => {
        const user = await db.findUser(parseInt(request.params.id));
        if (!user) {
            return reply.status(404).send({ error: 'Not found' });
        }
        return user;  // automatically serialized using the schema
    },
);

await fastify.listen({ port: 3000, host: '0.0.0.0' });
```

## 3. Plugin Architecture

Fastify's plugin system encapsulates routes, hooks, and decorators:

```typescript
import fp from 'fastify-plugin';

// Database plugin
async function databasePlugin(fastify) {
    const pool = await createPool(process.env.DATABASE_URL);
    fastify.decorate('db', pool);  // add to fastify instance

    fastify.addHook('onClose', async () => {
        await pool.end();  // cleanup on shutdown
    });
}

// Auth plugin
async function authPlugin(fastify) {
    fastify.addHook('preHandler', async (request, reply) => {
        const token = request.headers.authorization?.replace('Bearer ', '');
        if (!token) {
            return reply.status(401).send({ error: 'Unauthorized' });
        }
        request.user = await verifyToken(token);
    });
}

// Register plugins
fastify.register(fp(databasePlugin));   // fp() shares context with parent
fastify.register(fp(authPlugin));

// Route plugin (encapsulated — its hooks don't leak)
fastify.register(async (instance) => {
    instance.get('/admin/stats', async (request) => {
        return instance.db.query('SELECT COUNT(*) FROM users');
    });
}, { prefix: '/api/v1' });
```

## 4. Fastify vs. NestJS vs. Express

```
FASTIFY: Best raw performance, excellent TypeScript, plugin system
         Good for: high-throughput APIs, microservices, performance-critical services

NESTJS: Opinionated architecture (Angular-style), dependency injection
        Good for: large teams, complex applications, existing Angular teams

EXPRESS: Mature ecosystem, everyone knows it, lower performance
         Good for: existing codebases, simple APIs, maximum ecosystem compatibility

THROUGHPUT (approximate, plain JSON):
  Fastify:  ~80,000 req/s
  Express:  ~30,000 req/s
  NestJS:   ~45,000 req/s (on top of Fastify adapter)
```

## 5. Senior Engineer Perspective

**Fastify's JSON schema for security:** The response schema doesn't just optimize serialization — it also prevents accidental data leaks. If your schema says `{ id, name, email }`, and your DB returns `{ id, name, email, password_hash, internal_notes }`, Fastify strips the extra fields automatically. This is a security feature, not just a performance one.

**When to migrate from Express to Fastify:** If your API is CPU-bound on JSON serialization (visible in profiler as `JSON.stringify`), Fastify's schema-based serialization is a drop-in win. If your bottleneck is the database, the framework choice doesn't matter.
