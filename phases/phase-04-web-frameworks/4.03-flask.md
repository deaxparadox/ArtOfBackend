## 1. What Is Flask?

Flask (2010, Armin Ronacher) is a "microframework" — it provides only the essentials: URL routing, request/response handling, and a templating engine. Everything else (ORM, auth, validation, admin) is a plugin you choose.

**Flask's philosophy:** Start simple, add what you need. No magic, no enforced structure, no decisions made for you.

**When Flask was the standard:** 2010-2018, Flask dominated Python API development. FastAPI's 2018 launch shifted new projects, but millions of Flask applications are in production.

## 2. Flask Core

```python
from flask import Flask, request, jsonify, g
from functools import wraps
import jwt

app = Flask(__name__)

# Simple GET route
@app.route("/users/<int:user_id>")
def get_user(user_id: int):
    user = db.session.get(User, user_id)
    if not user:
        return jsonify({"error": "Not found"}), 404
    return jsonify({"id": user.id, "email": user.email})

# POST with request body
@app.route("/users", methods=["POST"])
def create_user():
    data = request.get_json()
    if not data:
        return jsonify({"error": "JSON body required"}), 400

    # Manual validation (Flask doesn't have Pydantic built in)
    if "email" not in data:
        return jsonify({"error": "email is required"}), 422

    user = User(email=data["email"], name=data.get("name", ""))
    db.session.add(user)
    db.session.commit()
    return jsonify({"id": user.id}), 201

# Decorator-based auth
def require_auth(f):
    @wraps(f)
    def decorated(*args, **kwargs):
        token = request.headers.get("Authorization", "").replace("Bearer ", "")
        try:
            payload = jwt.decode(token, app.config["SECRET_KEY"], algorithms=["HS256"])
            g.current_user_id = payload["sub"]
        except jwt.InvalidTokenError:
            return jsonify({"error": "Invalid token"}), 401
        return f(*args, **kwargs)
    return decorated

@app.route("/profile")
@require_auth
def get_profile():
    user = db.session.get(User, g.current_user_id)
    return jsonify({"id": user.id, "email": user.email})
```

## 3. Flask Application Factory Pattern

```python
# app/__init__.py — the factory pattern (production standard)
from flask import Flask
from flask_sqlalchemy import SQLAlchemy
from flask_migrate import Migrate

db = SQLAlchemy()
migrate = Migrate()

def create_app(config_name: str = "production") -> Flask:
    app = Flask(__name__)
    app.config.from_object(config[config_name])

    # Initialize extensions
    db.init_app(app)
    migrate.init_app(app, db)

    # Register blueprints (route grouping)
    from .routes.users import users_bp
    from .routes.orders import orders_bp
    app.register_blueprint(users_bp, url_prefix="/api/v1/users")
    app.register_blueprint(orders_bp, url_prefix="/api/v1/orders")

    # Error handlers
    @app.errorhandler(404)
    def not_found(e):
        return jsonify({"error": "Not found"}), 404

    @app.errorhandler(500)
    def internal_error(e):
        return jsonify({"error": "Internal error"}), 500

    return app

# wsgi.py
from app import create_app
app = create_app()
```

## 4. Flask vs. FastAPI

```
FLASK:
  ✓ More mature ecosystem (10 years of extensions)
  ✓ Simpler mental model (no async, no DI system)
  ✓ Better for legacy codebases
  ✓ Synchronous by default (simpler for new developers)
  ✓ Werkzeug under the hood (very debuggable)

  ✗ No auto-generated API docs
  ✗ No built-in validation (need marshmallow, flask-pydantic)
  ✗ Synchronous only by default (async possible but bolted on)
  ✗ More boilerplate for complex apps

FASTAPI:
  ✓ Auto docs (OpenAPI/Swagger)
  ✓ Pydantic validation built in
  ✓ Async native
  ✓ Better type safety
  ✓ Dependency injection

  ✗ Newer (less ecosystem depth)
  ✗ Async mental model required

RECOMMENDATION:
  New project → FastAPI (unless you specifically need Flask's simplicity)
  Existing Flask project → stay on Flask, consider gradual migration
  Teaching → Flask (simpler for beginners)
```

## 5. Senior Engineer Perspective

**Flask's lasting contribution:** It proved that Python web frameworks don't need to be "batteries included" to be excellent. The plugin ecosystem (Flask-SQLAlchemy, Flask-Login, Flask-Migrate, Flask-CORS) showed that composability can work as well as the monolithic Django approach.

**When to choose Flask in 2024:** If your team knows Flask well and the project doesn't require async or OpenAPI docs. Otherwise, FastAPI has superseded it for new API development. Flask is now the "simpler alternative to Django for sync apps" — a valid but narrowing niche.
