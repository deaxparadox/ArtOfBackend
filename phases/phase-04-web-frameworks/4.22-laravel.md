## Laravel (4.22)

Laravel (2011, Taylor Otwell) is the dominant PHP framework and widely considered the best developer experience in the PHP ecosystem. It borrows Rails' conventions and adds its own innovations.

```php
// Laravel routes and controllers
// routes/api.php
Route::middleware('auth:sanctum')->group(function () {
    Route::apiResource('users', UserController::class);
    Route::post('orders/{order}/cancel', [OrderController::class, 'cancel']);
});

// app/Http/Controllers/UserController.php
class UserController extends Controller
{
    public function index(Request $request): UserCollection
    {
        $users = User::query()
            ->when($request->search, fn($q, $s) => $q->search($s))
            ->with('profile')  // eager load
            ->paginate(20);
        return new UserCollection($users);  // API resource collection
    }

    public function store(CreateUserRequest $request): UserResource
    {
        $user = User::create($request->validated());
        event(new UserCreated($user));  // fire event
        return new UserResource($user);
    }
}

// app/Http/Requests/CreateUserRequest.php — automatic validation
class CreateUserRequest extends FormRequest
{
    public function rules(): array
    {
        return [
            'email' => 'required|email|unique:users',
            'name'  => 'required|string|min:1|max:100',
        ];
    }
}

// app/Models/User.php — Eloquent ORM
class User extends Model
{
    protected $fillable = ['email', 'name'];
    
    public function orders(): HasMany
    {
        return $this->hasMany(Order::class);
    }
    
    public function scopeActive(Builder $query): void
    {
        $query->where('active', true);
    }
}
```

**Laravel's best features:**
- `artisan` CLI: code generation, migrations, queue management
- Eloquent ORM: the best ORM in PHP, inspired by ActiveRecord
- Laravel Sanctum / Passport: API authentication out of the box
- Queues: first-class background job support (Redis, database, SQS)
- Telescope: developer debugging dashboard
- Forge: PaaS deployment platform designed for Laravel

## ASP.NET Core (4.23)

ASP.NET Core (2016, Microsoft) is a high-performance, cross-platform web framework that rivals Node.js and Go in benchmarks while providing the rich C# ecosystem.

```csharp
// Minimal API (modern style, ASP.NET Core 6+)
var builder = WebApplication.CreateBuilder(args);
builder.Services.AddDbContext<AppDbContext>();
builder.Services.AddScoped<IUserService, UserService>();
builder.Services.AddAuthentication().AddJwtBearer();
builder.Services.AddAuthorization();
builder.Services.AddEndpointsApiExplorer();
builder.Services.AddSwaggerGen();

var app = builder.Build();

app.UseAuthentication();
app.UseAuthorization();
app.UseSwagger(); app.UseSwaggerUI();

// Route handlers with DI
app.MapGet("/users/{id:int}", async (int id, IUserService svc) =>
    await svc.GetByIdAsync(id) is UserDto user
        ? Results.Ok(user)
        : Results.NotFound())
    .RequireAuthorization();

app.MapPost("/users",
    async ([FromBody] CreateUserRequest req, IUserService svc) =>
    {
        var user = await svc.CreateAsync(req);
        return Results.Created($"/users/{user.Id}", user);
    })
    .RequireAuthorization("AdminPolicy");

app.Run();
```

**ASP.NET Core's strengths:**
- Among the fastest web frameworks in any language (TechEmpower top 5)
- Excellent built-in support: DI, logging, health checks, auth, OpenAPI
- Native AOT in .NET 8: fast startup like Go, no JVM overhead
- Best-in-class Entity Framework Core ORM

## Axum (4.24)

Axum (2021, Tokio team) is Rust's most ergonomic async web framework, built on Tokio (the Rust async runtime) and Tower (service abstractions).

```rust
use axum::{
    extract::{Path, State},
    http::StatusCode,
    routing::{get, post},
    Json, Router,
};
use serde::{Deserialize, Serialize};
use sqlx::PgPool;

#[derive(Serialize, Deserialize)]
struct User { id: i32, name: String, email: String }

#[derive(Deserialize)]
struct CreateUser { name: String, email: String }

async fn get_user(
    Path(id): Path<i32>,
    State(pool): State<PgPool>,
) -> Result<Json<User>, StatusCode> {
    sqlx::query_as!(User, "SELECT id, name, email FROM users WHERE id=$1", id)
        .fetch_optional(&pool)
        .await
        .map_err(|_| StatusCode::INTERNAL_SERVER_ERROR)?
        .map(Json)
        .ok_or(StatusCode::NOT_FOUND)
}

async fn create_user(
    State(pool): State<PgPool>,
    Json(payload): Json<CreateUser>,
) -> Result<(StatusCode, Json<User>), StatusCode> {
    let user = sqlx::query_as!(
        User,
        "INSERT INTO users (name, email) VALUES ($1, $2) RETURNING id, name, email",
        payload.name, payload.email
    )
    .fetch_one(&pool)
    .await
    .map_err(|_| StatusCode::INTERNAL_SERVER_ERROR)?;

    Ok((StatusCode::CREATED, Json(user)))
}

#[tokio::main]
async fn main() {
    let pool = PgPool::connect(&std::env::var("DATABASE_URL").unwrap()).await.unwrap();

    let app = Router::new()
        .route("/users/:id", get(get_user))
        .route("/users", post(create_user))
        .with_state(pool);

    let listener = tokio::net::TcpListener::bind("0.0.0.0:8080").await.unwrap();
    axum::serve(listener, app).await.unwrap();
}
```

**Axum's strengths:** Type-safe extractors, compile-time checked SQL with SQLx, zero-cost abstractions. **Best choice if you're writing Rust for your backend.**
