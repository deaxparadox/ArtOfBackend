## 1. What Is Litestar?

Litestar (formerly Starlite, 2021) is a FastAPI alternative that shares its philosophy but makes different design choices. It is also ASGI-based and Pydantic-compatible, but adds:

- Built-in OpenTelemetry integration
- More opinionated project structure
- Native DTO (Data Transfer Object) layer
- Built-in rate limiting, caching, response compression
- Slightly more verbose but more explicit than FastAPI

```python
from litestar import Litestar, get, post
from litestar.di import Provide
from litestar.config.cors import CORSConfig

@get("/users/{user_id:int}")
async def get_user(user_id: int) -> dict:
    return {"id": user_id, "name": "Alice"}

@post("/users")
async def create_user(data: CreateUserRequest) -> UserResponse:
    user = await db.create(data)
    return UserResponse.from_orm(user)

app = Litestar(
    route_handlers=[get_user, create_user],
    cors_config=CORSConfig(allow_origins=["*"]),
    dependencies={"db": Provide(get_db)},
)
```

## 2. Litestar vs. FastAPI

```
LITESTAR ADVANTAGES:
  ✓ More batteries included (caching, rate limiting built-in)
  ✓ Stricter typing enforcement
  ✓ Better plugin architecture
  ✓ OpenTelemetry built-in

FASTAPI ADVANTAGES:
  ✓ Larger community (10x more users)
  ✓ More tutorials, Stack Overflow answers
  ✓ Better ecosystem compatibility
  ✓ More widely deployed

RECOMMENDATION:
  FastAPI for nearly all new projects.
  Litestar worth considering if you want stricter conventions
  or its built-in features (and are OK with smaller community).
```
