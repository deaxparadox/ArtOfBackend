## 1. Go's Standard Library HTTP

Go's `net/http` package is production-grade out of the box — no framework required for many use cases. Netflix, Google, and Cloudflare run Go services with minimal or no framework beyond stdlib.

```go
package main

import (
    "encoding/json"
    "log/slog"
    "net/http"
    "strconv"
    "strings"
)

type User struct {
    ID    int    `json:"id"`
    Name  string `json:"name"`
    Email string `json:"email"`
}

func (s *Server) handleGetUser(w http.ResponseWriter, r *http.Request) {
    // Extract path parameter manually (net/http 1.22 has path params)
    parts := strings.Split(r.URL.Path, "/")
    id, err := strconv.Atoi(parts[len(parts)-1])
    if err != nil {
        http.Error(w, "Invalid ID", http.StatusBadRequest)
        return
    }

    user, err := s.db.GetUser(r.Context(), id)
    if err != nil {
        http.Error(w, "Not found", http.StatusNotFound)
        return
    }

    w.Header().Set("Content-Type", "application/json")
    json.NewEncoder(w).Encode(user)
}

func main() {
    mux := http.NewServeMux()
    
    // Go 1.22+: built-in path parameters and method-specific routing
    mux.HandleFunc("GET /users/{id}", handleGetUser)
    mux.HandleFunc("POST /users", handleCreateUser)
    mux.HandleFunc("DELETE /users/{id}", handleDeleteUser)

    server := &http.Server{
        Addr:         ":8080",
        Handler:      mux,
        ReadTimeout:  5 * time.Second,
        WriteTimeout: 10 * time.Second,
        IdleTimeout:  120 * time.Second,
    }

    slog.Info("Starting server", "addr", ":8080")
    log.Fatal(server.ListenAndServe())
}
```

**Go 1.22 (Feb 2024) added method-based routing and path parameters directly to `net/http`.** Before 1.22, you needed a router library for this. After 1.22, simple APIs can be built entirely on stdlib.

**When to use just `net/http`:**
- Simple internal APIs with < 10 routes
- Maximum control and minimal dependencies
- When you want to understand exactly what's happening
