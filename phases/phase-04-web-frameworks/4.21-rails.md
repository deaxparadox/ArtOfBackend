## 1. What Is Rails?

Ruby on Rails (2004, DHH) is the framework that shaped modern web development. Its conventions — MVC, REST, ActiveRecord, migrations, scaffolding — are now the default mental model for web frameworks in every language. FastAPI, NestJS, Laravel, Django — all borrowed ideas from Rails.

**Rails at scale:** Shopify processes $5.6B on Black Friday on Rails. GitHub ran on Rails for years. Basecamp (where Rails was born) still runs on Rails with a tiny team serving millions of users.

## 2. The Rails Way

```ruby
# Rails generator creates everything:
# rails generate scaffold User email:string name:string
# Creates: model, migration, controller, views, routes, tests

# config/routes.rb
Rails.application.routes.draw do
  resources :users              # GET/POST /users, GET/PUT/DELETE /users/:id
  resources :orders do
    member do
      post :cancel              # POST /orders/:id/cancel
    end
    collection do
      get :pending              # GET /orders/pending
    end
  end
  namespace :api do
    namespace :v1 do
      resources :users, only: [:index, :show, :create]
    end
  end
end

# app/models/user.rb
class User < ApplicationRecord
  has_many :orders, dependent: :destroy
  has_one :profile
  
  validates :email, presence: true, uniqueness: true, format: { with: URI::MailTo::EMAIL_REGEXP }
  validates :name, presence: true, length: { minimum: 1, maximum: 100 }
  
  before_create :generate_api_key
  after_create :send_welcome_email
  
  scope :active, -> { where(active: true) }
  scope :recent, -> { order(created_at: :desc).limit(20) }
  
  private
  
  def generate_api_key
    self.api_key = SecureRandom.hex(32)
  end
  
  def send_welcome_email
    UserMailer.welcome(self).deliver_later  # async, uses background jobs
  end
end

# app/controllers/api/v1/users_controller.rb
module Api
  module V1
    class UsersController < ApplicationController
      before_action :authenticate!
      before_action :set_user, only: [:show, :update, :destroy]
      
      def index
        @users = User.active.recent.page(params[:page]).per(20)
        render json: UserSerializer.new(@users).serializable_hash
      end
      
      def show
        render json: UserSerializer.new(@user).serializable_hash
      end
      
      def create
        @user = User.new(user_params)
        if @user.save
          render json: UserSerializer.new(@user).serializable_hash, status: :created
        else
          render json: { errors: @user.errors }, status: :unprocessable_entity
        end
      end
      
      private
      
      def set_user
        @user = User.find(params[:id])
      end
      
      def user_params
        params.require(:user).permit(:email, :name)
      end
    end
  end
end
```

## 3. ActiveRecord: The ORM Rails Made Famous

```ruby
# ActiveRecord: the ORM pattern Rails popularized
# Every ORM in every language was influenced by this

# Queries build up lazily:
User.where(active: true)               # SELECT * FROM users WHERE active = true
    .where("created_at > ?", 30.days.ago)
    .includes(:orders)                 # eager load to prevent N+1
    .order(created_at: :desc)
    .limit(20)
    .offset(params[:offset].to_i)

# Associations:
user = User.find(1)
user.orders                            # User has_many :orders
user.orders.where(status: "pending")   # scoped
user.orders.create(total: 5000)        # create via association

# Migrations:
class AddPhoneToUsers < ActiveRecord::Migration[7.1]
  def change
    add_column :users, :phone, :string
    add_index  :users, :phone, unique: true
  end
end
```

## 5. Senior Engineer Perspective

**Rails' lasting lesson:** Convention over configuration is a genuine productivity win. When everyone follows Rails conventions, you spend zero time on "what folder does this go in?" or "what's this class called?" — those questions are answered by the framework. This allows small teams to punch above their weight.

**The Rails scaling path:** Start with a monolith (Rails is perfect for this). Extract services as needed (Rails has excellent API mode for this). Shopify's architecture is instructive: Rails core with microservices extracted for the highest-load paths.
