## RBAC (8.17): Role-Based Access Control

Assign roles to users; roles have permissions.

```python
# Simple RBAC implementation
from enum import Enum
from typing import set

class Permission(str, Enum):
    READ_USERS = "read:users"
    WRITE_USERS = "write:users"
    DELETE_USERS = "delete:users"
    READ_ORDERS = "read:orders"
    ADMIN = "admin"

ROLE_PERMISSIONS: dict[str, set[Permission]] = {
    "viewer":   {Permission.READ_USERS, Permission.READ_ORDERS},
    "editor":   {Permission.READ_USERS, Permission.WRITE_USERS, Permission.READ_ORDERS},
    "admin":    set(Permission),  # all permissions
}

def has_permission(user_roles: list[str], required: Permission) -> bool:
    for role in user_roles:
        if required in ROLE_PERMISSIONS.get(role, set()):
            return True
    return False

# FastAPI dependency
from fastapi import Depends, HTTPException

def require_permission(permission: Permission):
    async def check(user: dict = Depends(get_current_user)):
        if not has_permission(user.get("roles", []), permission):
            raise HTTPException(403, f"Permission required: {permission}")
        return user
    return check

# Usage
@app.delete("/users/{user_id}")
async def delete_user(
    user_id: int,
    admin = Depends(require_permission(Permission.DELETE_USERS))
):
    await db.delete_user(user_id)
```

## ABAC (8.18): Attribute-Based Access Control

ABAC policies evaluate attributes of the user, resource, and environment:

```python
# ABAC: flexible policy expressions
from dataclasses import dataclass

@dataclass
class AccessContext:
    user_id: int
    user_roles: list[str]
    resource_owner_id: int
    resource_type: str
    action: str
    environment: dict  # time, IP, etc.

def can_access(ctx: AccessContext) -> bool:
    """Policy evaluation (can be as complex as needed)."""

    # Admins can do anything
    if "admin" in ctx.user_roles:
        return True

    # Users can only modify their own resources
    if ctx.action in ["update", "delete"]:
        if ctx.user_id != ctx.resource_owner_id:
            return False

    # Time-based restriction: no writes after business hours
    import datetime
    if ctx.action in ["create", "update", "delete"]:
        hour = datetime.datetime.now().hour
        if not (9 <= hour <= 18):
            if "after_hours_access" not in ctx.user_roles:
                return False

    return True
```

## ReBAC (8.19): Relationship-Based Access Control

Google Zanzibar (2019) paper — the authorization model behind Google Drive, Google Docs:

```
ZANZIBAR MODEL:
  Tuple: (object, relation, user)
  Examples:
    (document:1, viewer, user:alice)
    (document:1, editor, group:engineers)
    (folder:reports, owner, user:bob)

  Rule: if user has viewer on folder:reports,
        they also have viewer on all documents in that folder.

QUERY: "Can alice edit document:1?"
  → Check: (document:1, editor, user:alice) → YES
  OR
  → Check: (document:1, editor, group:X) where alice is member of group:X
  OR
  → Inherit from parent folder

IMPLEMENTATIONS:
  SpiceDB (open source Zanzibar): most popular
  Ory Keto: open source, simpler
  Auth0 FGA (Fine-Grained Authorization): managed
  Google Zanzibar: not public, but SpiceDB replicates it
```

## OPA (Open Policy Agent) (8.20)

OPA is a general-purpose policy engine. Write policies in Rego language, evaluate them as code:

```rego
# policy.rego
package authz

# Allow if user is admin
allow {
    input.user.role == "admin"
}

# Allow reading own profile
allow {
    input.action == "read"
    input.resource.type == "user"
    input.resource.id == input.user.id
}

# Allow managers to read their reports' data
allow {
    input.action == "read"
    input.resource.type == "report"
    input.user.manages[_] == input.resource.owner_id
}
```

```python
# Query OPA from Python
import httpx

async def check_policy(user: dict, action: str, resource: dict) -> bool:
    response = await httpx.post(
        "http://opa:8181/v1/data/authz/allow",
        json={"input": {"user": user, "action": action, "resource": resource}},
    )
    return response.json().get("result", False)
```

## Row-Level Security in PostgreSQL (8.21)

```sql
-- Enable RLS on table
ALTER TABLE documents ENABLE ROW LEVEL SECURITY;

-- Policy: users can only see their own documents
CREATE POLICY user_documents ON documents
    USING (owner_id = current_setting('app.current_user_id')::bigint);

-- Policy: admins see everything
CREATE POLICY admin_all ON documents
    USING (current_setting('app.is_admin')::boolean = true);

-- Set session variables from application
-- (set per-request before queries)
SET app.current_user_id = '123';
SET app.is_admin = 'false';
SELECT * FROM documents;
-- Only returns documents where owner_id = 123
```
