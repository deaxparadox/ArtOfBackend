## 1. What Is It? (Theory)

Meilisearch (2018, Quentin de Quelen, written in Rust) is an open-source search engine focused on instant, as-you-type search with a superb developer experience. Like Typesense, it targets the "add search to my app in an afternoon" use case rather than the "run a search cluster in production" use case that Elasticsearch targets.

**What sets Meilisearch apart:**

- Written in Rust: exceptional performance, low memory usage
- Out-of-the-box features: typo tolerance, synonyms, stop words, highlighting, geosearch
- Zero-config relevance: sensible defaults that work for most use cases
- Excellent documentation and SDKs
- Multilingual search built-in

## 2. Core Features

### Setup and Indexing

```python
import meilisearch

client = meilisearch.Client("http://127.0.0.1:7700", "masterKey")

# Create index (schema-less by default, inferred from documents)
index = client.index("products")

# Bulk index
index.add_documents([
    {
        "id": 1,
        "name": "Nike Air Max Running Shoes",
        "description": "Lightweight running shoes with maximum cushioning",
        "price_cents": 12999,
        "category": "footwear",
        "brand": "Nike",
        "in_stock": True,
        "_geo": {"lat": 40.7128, "lng": -74.0060}  # geosearch
    },
    # ...
])

# Configure index settings
index.update_settings({
    # Filterable attributes (for filter/facet)
    "filterableAttributes": ["category", "brand", "price_cents", "in_stock"],
    
    # Sortable attributes  
    "sortableAttributes": ["price_cents", "created_at"],
    
    # Searchable attributes (with importance order)
    "searchableAttributes": ["name", "description", "brand"],
    # Order matters: name matches score higher than description matches
    
    # Synonyms
    "synonyms": {
        "sneakers": ["shoes", "trainers", "footwear"],
        "sofa": ["couch", "settee"],
    },
    
    # Stop words
    "stopWords": ["the", "a", "an"],
    
    # Typo tolerance
    "typoTolerance": {
        "enabled": True,
        "minWordSizeForTypos": {
            "oneTypo": 5,    # 5+ char words: allow 1 typo
            "twoTypos": 9,   # 9+ char words: allow 2 typos
        }
    }
})
```

### Searching

```python
# Basic search
results = index.search("runnning shoes")  # typo tolerated
# Returns: hits, processingTimeMs, query, nbHits

# Advanced search
results = index.search(
    "running shoes",
    {
        "filter": ["in_stock = true", "price_cents < 20000"],
        "sort": ["price_cents:asc"],
        "facets": ["category", "brand"],
        "attributesToHighlight": ["name", "description"],
        "highlightPreTag": "<em>",
        "highlightPostTag": "</em>",
        "limit": 20,
        "offset": 0,
    }
)

# Geosearch: find products near a location
results = index.search(
    "running shoes",
    {
        "filter": ["_geoRadius(40.7128, -74.0060, 10000)"],  # 10km radius
        "sort": ["_geoPoint(40.7128, -74.0060):asc"],         # sort by distance
    }
)
```

## 3. Meilisearch vs. Typesense

```
MEILISEARCH:
  ✓ Written in Rust (faster, lower memory)
  ✓ Better multilingual support (language detection, locale-specific analysis)
  ✓ More opinionated defaults (less configuration = faster to production)
  ✓ Excellent documentation
  ✗ Hosted solution (Meilisearch Cloud) is newer than Typesense Cloud
  ✗ Slightly less mature distributed/multi-node story

TYPESENSE:
  ✓ More mature distributed setup
  ✓ Typesense Cloud has been available longer
  ✓ Slightly more control over relevance tuning
  ✗ Written in C++ (still fast, but Meilisearch edges it out in benchmarks)

PRACTICAL DIFFERENCE:
  Both are excellent for the same use cases.
  Choose Typesense if cloud/distributed deployment is priority.
  Choose Meilisearch if developer experience and multilingual are priority.
  Either is dramatically simpler than Elasticsearch for simple search.
```

## 4. Senior Engineer Perspective

**The instant-search architecture:**

Meilisearch is designed for "search as you type" — every keystroke triggers a new search. This requires sub-10ms response times. To achieve this, Meilisearch pre-computes its data structures (LMDB database, not inverted index) to make lookups extremely fast at query time. The trade-off: indexing is slower than Elasticsearch, but queries are consistently fast without tuning.

For a documentation site search, blog search, or small-to-medium e-commerce catalog (<5M documents): Meilisearch or Typesense deployed as a sidecar alongside your main application is often the simplest complete solution available.
