## 1. What Is It? (Theory)

Algolia (2012, Nicolas Dessaigne and Julien Lemoine) is the leading managed search-as-a-service platform. You send your data to Algolia's API, they handle all infrastructure, and you get instant, relevant search via their globally distributed network.

**What makes Algolia the premium choice:**

- **Global PoP network:** search served from 100+ locations worldwide (< 5ms latency globally)
- **InstantSearch SDKs:** React, Vue, Angular, iOS, Android components out of the box
- **NeuralSearch:** hybrid keyword + vector search built-in (no ML expertise needed)
- **Analytics dashboard:** zero-result queries, click analytics, conversion tracking
- **A/B testing:** test ranking strategies with statistical significance

Algolia is chosen when developer productivity and user experience are prioritized over infrastructure control and cost.

## 2. The Algolia Pricing Model

```
PRICING (approximate 2024):
  Free tier: 10,000 records + 100,000 requests/month
  Pay-as-you-go:
    Records:  ~$0.50 per 1,000 records/month
    Requests: ~$1.50 per 1,000 search requests
  
  COST EXAMPLE: 100,000 products, 1M searches/month
    Records:  100 × $0.50 = $50/month
    Requests: 1,000 × $1.50 = $1,500/month
    Total: ~$1,550/month
  
  COMPARISON:
    Typesense Cloud (same scale): ~$50-100/month
    Self-hosted Elasticsearch: ~$100-300/month (server cost)
    Algolia: ~$1,550/month (premium for zero-ops, global network)
  
  WHO PAYS ALGOLIA'S PRICE:
    Teams where search is product-critical (user experience > cost)
    Startups that can't afford search expertise
    E-commerce where 1% better search = significant revenue
    Companies where developer time > infrastructure cost
```

## 3. Algolia InstantSearch

Algolia's killer feature is their InstantSearch UI component library — production-ready search UI in minutes:

```jsx
// React example: full search UI in ~30 lines
import algoliasearch from 'algoliasearch';
import { InstantSearch, SearchBox, Hits, RefinementList,
         Pagination, Highlight, Stats } from 'react-instantsearch';

const searchClient = algoliasearch(
    'YOUR_APP_ID',
    'YOUR_SEARCH_API_KEY'  // read-only key, safe in frontend
);

function ProductSearch() {
    return (
        <InstantSearch searchClient={searchClient} indexName="products">
            <SearchBox />          {/* search input with debouncing */}
            <Stats />              {/* "123 results in 4ms" */}
            
            <div className="filters">
                {/* Faceted filters with checkboxes */}
                <RefinementList attribute="category" />
                <RefinementList attribute="brand" />
            </div>
            
            <Hits hitComponent={({ hit }) => (
                <div>
                    <img src={hit.image} />
                    <Highlight attribute="name" hit={hit} />  {/* highlighted snippet */}
                    <p>${hit.price_cents / 100}</p>
                </div>
            )} />
            
            <Pagination />
        </InstantSearch>
    );
}
// This is a complete, production-ready search UI.
// Zero custom search logic. Handles: debouncing, loading states, 
// routing (URL params update as filters change), mobile.
```

## 4. Algolia vs. Typesense vs. Elasticsearch

```
                    Algolia         Typesense       Elasticsearch
Operations:         Zero (SaaS)     Low             High
Setup time:         Minutes         1-2 hours       Days
Global latency:     <5ms globally   Depends on PoP  Depends on deployment
UI components:      Full library    Basic           None built-in
Analytics:          Built-in        Basic           Via Kibana
Cost (1M req/mo):   ~$1,500        ~$100           ~$150 (hosting)
Data privacy:       Your data at Algolia  Self-hosted    Self-hosted
Scale ceiling:      Effectively ∞  Medium-large    Very large

CHOOSE ALGOLIA WHEN:
  ✓ Speed to market is critical (search in production today)
  ✓ Global user base needing consistent low latency
  ✓ Team lacks search infrastructure expertise
  ✓ Search is core to user experience (conversion rates matter)
  ✓ Budget allows premium pricing

CHOOSE TYPESENSE/MEILISEARCH WHEN:
  ✓ Cost is a primary concern
  ✓ Data privacy requires self-hosting
  ✓ Usage pattern where per-request pricing would be expensive
  ✓ Similar feature set at 10-30x lower cost is acceptable

CHOOSE ELASTICSEARCH WHEN:
  ✓ Massive scale, complex analytics
  ✓ Log/observability use case
  ✓ Need full control over infrastructure
```

## 5. Senior Engineer Perspective

**The Algolia cost cliff:**

At low traffic, Algolia is affordable and the developer experience is extraordinary. At high traffic (millions of searches/day), the per-request pricing becomes painful. Many teams start on Algolia, grow into significant costs, and migrate to Typesense or self-hosted search at scale. Plan this transition early — migration is doable but takes engineering time.

**The search-as-a-service data consideration:**

Sending your product catalog to Algolia means your data lives in their infrastructure. For most e-commerce: acceptable. For sensitive domains (healthcare, finance, legal): evaluate data sovereignty requirements before choosing a managed search provider.
