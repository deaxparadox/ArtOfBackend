## 1. What Is It? (Theory)

OpenSearch (Amazon, 2021) is a community-driven, open-source fork of Elasticsearch 7.10. It was created when Elastic changed Elasticsearch's license from open-source (Apache 2.0) to a proprietary "SSPL" license in January 2021, which AWS determined was incompatible with their managed service business.

**The backstory:**

Elastic's change meant AWS could no longer legally modify and redistribute Elasticsearch in their managed Amazon OpenSearch Service (formerly Amazon Elasticsearch Service). Amazon forked Elasticsearch 7.10 (the last Apache-licensed version) and created OpenSearch — maintaining it as a genuinely open-source (Apache 2.0) project.

## 2. OpenSearch vs. Elasticsearch

```
API COMPATIBILITY:
  OpenSearch maintains near-identical API compatibility with Elasticsearch 7.x.
  The Elasticsearch Python, Java, and Go clients work with OpenSearch.
  Most queries, mappings, and index settings are identical.

DIVERGENCE OVER TIME:
  OpenSearch has developed unique features not in Elasticsearch:
  ✓ OpenSearch Dashboards (fork of Kibana, Apache 2.0 licensed)
  ✓ SQL plugin improvements
  ✓ k-NN (k-nearest neighbor) for vector search
  ✓ Security plugin (built-in, not commercial tier)
  ✓ Anomaly detection built-in
  
  Elasticsearch has continued developing features not in OpenSearch:
  ✓ Elasticsearch Serverless
  ✓ ELSER (semantic search model)
  ✓ Some performance optimizations

LICENSE:
  OpenSearch: Apache 2.0 (fully open source)
  Elasticsearch 8.x+: Elastic License 2.0 (ELv2) — not truly open source
                      (cannot build competing services on top of it)

MANAGED SERVICES:
  AWS OpenSearch Service: managed OpenSearch (and legacy Elasticsearch 7.x)
  Elastic Cloud: managed Elasticsearch by Elastic (higher cost, latest features)
```

## 3. When to Choose Which

```
CHOOSE OPENSEARCH WHEN:
  ✓ Deploying on AWS (Amazon OpenSearch Service is a natural fit)
  ✓ You need Apache 2.0 licensing for legal/compliance reasons
  ✓ You want a fully open-source solution with no commercial restrictions
  ✓ Built-in security (no commercial tier needed for TLS, auth, RBAC)
  ✓ Cost: managed OpenSearch is generally cheaper than Elastic Cloud

CHOOSE ELASTICSEARCH WHEN:
  ✓ You need the latest Elasticsearch features (ELv2 licensed but more current)
  ✓ ELSER (semantic search with Elastic's embedding model)
  ✓ Elastic's commercial support and roadmap matters to you
  ✓ GCP or Azure (Elastic Cloud has better multi-cloud support)
  ✓ Using Kibana specifically (not just dashboards)

THE PRACTICAL REALITY FOR MOST TEAMS:
  Both are excellent. The API is nearly identical.
  AWS teams: OpenSearch (natural ecosystem fit, cost-effective).
  Teams wanting cutting-edge semantic search: Elasticsearch.
  Teams needing enterprise support SLAs: Elastic Cloud.
  Open-source purists: OpenSearch.
```

## 4. Migrating Between Them

The APIs are similar enough that migration is straightforward for most use cases:

```python
# Elasticsearch Python client → OpenSearch Python client
# Install: pip install opensearch-py (not elasticsearch-py)

from opensearchpy import OpenSearch  # was: from elasticsearch import Elasticsearch

es = OpenSearch(
    hosts=[{"host": "my-opensearch.us-east-1.es.amazonaws.com", "port": 443}],
    http_auth=("admin", "password"),
    use_ssl=True,
    verify_certs=True,
)

# Queries are IDENTICAL to Elasticsearch
result = es.search(
    index="products",
    body={
        "query": {
            "bool": {
                "must": [{"match": {"description": "running shoes"}}],
                "filter": [{"term": {"in_stock": True}}]
            }
        }
    }
)
# Same response format as Elasticsearch
```

## 5. Senior Engineer Perspective

**The license change matters more than it seems:**

The 2021 license change by Elastic was contentious. Elasticsearch/Kibana are still used by millions, but the community is fragmented. Many large companies (including AWS, Red Hat, SAP, and others) backed OpenSearch precisely because of the licensing concern. When building a product on a search engine, consider: "Can I commercially distribute software that includes this?"

For most SaaS teams: the distinction doesn't matter (you're deploying, not redistributing). For ISVs embedding search into a product: OpenSearch's Apache 2.0 license is safer.
