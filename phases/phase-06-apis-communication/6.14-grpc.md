## Protocol Buffers (6.14)

Protocol Buffers (protobuf) is Google's binary serialization format — the foundation of gRPC. It is strongly typed, compact, and fast.

```protobuf
// user.proto — defines the schema
syntax = "proto3";

package user.v1;

// Message: like a struct/class definition
message User {
    int64 id = 1;             // field number (used in binary encoding, not name)
    string email = 2;
    string name = 3;
    repeated string tags = 4; // "repeated" = list
    UserStatus status = 5;
    google.protobuf.Timestamp created_at = 6;
}

enum UserStatus {
    USER_STATUS_UNSPECIFIED = 0;  // proto3: first value must be 0
    USER_STATUS_ACTIVE = 1;
    USER_STATUS_INACTIVE = 2;
}

message GetUserRequest {
    int64 user_id = 1;
}

message ListUsersResponse {
    repeated User users = 1;
    string next_cursor = 2;
    bool has_more = 3;
}

// Service: defines the RPC methods
service UserService {
    rpc GetUser(GetUserRequest) returns (User);
    rpc ListUsers(ListUsersRequest) returns (ListUsersResponse);
    rpc CreateUser(CreateUserRequest) returns (User);
    rpc StreamUserEvents(StreamUserEventsRequest) returns (stream UserEvent);  // server streaming
    rpc UploadUsers(stream User) returns (UploadResult);  // client streaming
    rpc Chat(stream ChatMessage) returns (stream ChatMessage);  // bidirectional streaming
}
```

**Why protobuf is faster than JSON:**

```
JSON:
  {"id": 123, "email": "alice@example.com", "name": "Alice"}
  → 52 bytes (text, field names repeated every time)
  → Parse: string parsing, key lookup

Protobuf:
  \x08\x7b\x12\x15alice@example.com\x1a\x05Alice
  → 27 bytes (binary, field numbers instead of names)
  → Parse: binary read, direct field assignment
  → 2-4x smaller, 3-10x faster to serialize/deserialize
```

**Field numbers:** Critical concept. When you change a protobuf schema, you NEVER change field numbers — old code reading data serialized with new schema uses the numbers to identify fields. Change the type or semantics but keep the same number = compatibility preserved.

## gRPC: HTTP/2-Based RPC (6.15)

gRPC (Google Remote Procedure Call) uses HTTP/2 for transport and Protocol Buffers for serialization. It generates client and server code in 11+ languages from the `.proto` file.

```python
# Python gRPC server (generated stubs from user.proto)
import grpc
from concurrent import futures
import user_pb2
import user_pb2_grpc

class UserServicer(user_pb2_grpc.UserServiceServicer):

    async def GetUser(self, request, context):
        user = await db.get_user(request.user_id)
        if not user:
            await context.abort(grpc.StatusCode.NOT_FOUND, f"User {request.user_id} not found")
        return user_pb2.User(
            id=user.id,
            email=user.email,
            name=user.name,
            status=user_pb2.USER_STATUS_ACTIVE,
        )

    async def StreamUserEvents(self, request, context):
        """Server streaming: yield multiple responses."""
        async for event in db.stream_user_events(request.user_id):
            if context.cancelled():  # client disconnected
                break
            yield user_pb2.UserEvent(
                event_type=event.type,
                timestamp=event.timestamp,
            )

# Start server
async def serve():
    server = grpc.aio.server()
    user_pb2_grpc.add_UserServiceServicer_to_server(UserServicer(), server)
    server.add_insecure_port('[::]:50051')
    await server.start()
    await server.wait_for_termination()
```

```python
# Python gRPC client
async def get_user(user_id: int):
    async with grpc.aio.insecure_channel('localhost:50051') as channel:
        stub = user_pb2_grpc.UserServiceStub(channel)
        try:
            user = await stub.GetUser(user_pb2.GetUserRequest(user_id=user_id))
            return user
        except grpc.aio.AioRpcError as e:
            if e.code() == grpc.StatusCode.NOT_FOUND:
                return None
            raise
```

## gRPC vs. REST (6.16)

```
DIMENSION        gRPC                        REST/HTTP
─────────────────────────────────────────────────────────
Transport        HTTP/2                      HTTP/1.1 or HTTP/2
Serialization    Protocol Buffers (binary)   JSON (text)
Contract         .proto file (strict)        OpenAPI (optional)
Latency          Lower (binary + HTTP/2)     Higher (text + often HTTP/1.1)
Payload size     2-4x smaller                Larger (field names in JSON)
Streaming        Native (4 modes)            Limited (SSE, WebSockets separate)
Browser support  Limited (needs grpc-web)    Full native support
Code generation  Automatic (11+ languages)   Optional (openapi-generator)
Type safety      Compile-time (proto types)  Runtime (optional with OpenAPI)
Human-readable   No (binary)                 Yes
Debugging        Harder (need tools)         Easy (curl, browser)
─────────────────────────────────────────────────────────

USE gRPC FOR:
  ✓ Internal service-to-service communication
  ✓ High-performance APIs (mobile → backend, data pipelines)
  ✓ Streaming (server push, bidirectional)
  ✓ Strongly-typed contract between services you control
  ✓ Polyglot microservices (Go service → Python service → Java service)

USE REST FOR:
  ✓ Public APIs (browser support required)
  ✓ When human readability matters
  ✓ Simple CRUD (REST is simpler to implement)
  ✓ When consumers may not have gRPC support

COMMON ARCHITECTURE:
  External clients → REST/GraphQL → Internal services → gRPC
  The public API uses REST (browsers, third parties).
  Internal microservices use gRPC (performance, type safety).
```

## gRPC-web (6.17)

Browsers can't use gRPC directly (HTTP/2 lacks some required features). gRPC-web is a proxy protocol:

```
Browser → HTTP/1.1 or HTTP/2 → Envoy/Nginx proxy → HTTP/2 gRPC → Backend

// JavaScript gRPC-web client
import { UserServiceClient } from './generated/user_grpc_web_pb';
import { GetUserRequest } from './generated/user_pb';

const client = new UserServiceClient('https://api.example.com');
const request = new GetUserRequest();
request.setUserId(123);

client.getUser(request, {}, (err, response) => {
    if (err) console.error(err);
    else console.log(response.toObject());
});
```

**Alternative:** ConnectRPC — a modern alternative to gRPC-web that works with browsers natively without a proxy, using standard HTTP/1.1 or HTTP/2.

## gRPC Interceptors (6.18)

Interceptors are gRPC's equivalent of middleware:

```python
# Python gRPC interceptor for authentication
import grpc

class AuthInterceptor(grpc.aio.ServerInterceptor):
    async def intercept_service(self, continuation, handler_call_details):
        # Extract metadata (gRPC equivalent of HTTP headers)
        metadata = dict(handler_call_details.invocation_metadata)
        token = metadata.get('authorization', '').replace('Bearer ', '')

        if not token:
            async def deny(request, context):
                await context.abort(grpc.StatusCode.UNAUTHENTICATED, 'Missing token')
            return grpc.unary_unary_rpc_method_handler(deny)

        # Verify token (pass user to handler via context)
        user = await verify_jwt(token)
        # Store in context for handlers to access
        return await continuation(handler_call_details)

# Add to server
server = grpc.aio.server(interceptors=[AuthInterceptor()])
```

## Buf: Modern Protobuf Toolchain (6.19)

```bash
# buf.yaml — project configuration
version: v1
lint:
  use: [DEFAULT]
  ignore: [google]
breaking:
  use: [FILE]    # detect breaking changes (changed field numbers, removed fields)

# Lint your .proto files
buf lint

# Detect breaking changes against previous version
buf breaking --against '.git#branch=main'

# Generate code from .proto files
buf generate

# buf.gen.yaml
version: v1
plugins:
  - plugin: python
    out: generated/
  - plugin: grpc-python
    out: generated/

# Buf Schema Registry (BSR): share proto schemas across teams
# Like npm but for .proto files
buf push --tag v1.0.0
```

**Buf vs. protoc:** `protoc` (the protobuf compiler) has a complex plugin system. Buf wraps it with dependency management, linting, breaking change detection, and a registry. Use Buf for any production gRPC project.
