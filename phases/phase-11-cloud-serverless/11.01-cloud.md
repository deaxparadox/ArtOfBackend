## AWS Core Services (11.1-11.20)

```
AWS ARCHITECTURE MENTAL MODEL:

COMPUTE:
  EC2:      Virtual machines (full control, pay by hour)
  ECS:      Docker containers (Fargate: serverless containers, EC2: you manage nodes)
  EKS:      Managed Kubernetes
  Lambda:   Serverless functions (pay per invocation)

STORAGE:
  S3:       Object storage (files, images, backups, static sites)
            Highly durable (11 nines), infinite scale
  EBS:      Block storage attached to EC2 (like a hard drive)
  EFS:      Network file system (shared across EC2 instances)

DATABASE:
  RDS:      Managed PostgreSQL/MySQL/MariaDB/Oracle/SQL Server
  Aurora:   AWS's own MySQL/Postgres-compatible, 5x faster than MySQL
  DynamoDB: Serverless NoSQL, massive scale
  ElastiCache: Managed Redis/Memcached

NETWORKING:
  VPC:      Your private network in AWS
  ALB/NLB:  Load balancers (L7/L4)
  CloudFront: CDN
  Route 53: DNS

MESSAGING:
  SQS:      Queue (decouple producers from consumers)
  SNS:      Pub/sub (fan-out to multiple subscribers)
  EventBridge: Event bus (route events to multiple targets)
  Kinesis:  Real-time streaming data (like Kafka, managed)

SECURITY:
  IAM:      Identity and access management (users, roles, policies)
  KMS:      Key Management Service (encrypt/decrypt)
  Secrets Manager: Store and rotate secrets
  WAF:      Web Application Firewall

MONITORING:
  CloudWatch: Metrics, logs, alarms
  X-Ray:    Distributed tracing
```

### AWS IAM: The Most Important AWS Service

```json
// IAM policy: principle of least privilege
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": [
                "s3:GetObject",
                "s3:PutObject"
            ],
            "Resource": "arn:aws:s3:::my-bucket/uploads/*"
            // Only access uploads folder, not entire bucket
        },
        {
            "Effect": "Allow",
            "Action": [
                "secretsmanager:GetSecretValue"
            ],
            "Resource": "arn:aws:secretsmanager:us-east-1:123456:secret:prod/myapp/*"
            // Only access this app's secrets, not all secrets
        }
    ]
}
```

## S3: Patterns and Best Practices

```python
import boto3
from botocore.exceptions import ClientError

s3 = boto3.client("s3")

# Upload file
s3.upload_fileobj(file_obj, "my-bucket", "uploads/user_123/avatar.jpg",
    ExtraArgs={
        "ContentType": "image/jpeg",
        "ServerSideEncryption": "AES256",  # encrypt at rest
    }
)

# Generate pre-signed URL (let clients upload directly to S3 — bypass your server)
url = s3.generate_presigned_url(
    "put_object",
    Params={
        "Bucket": "my-bucket",
        "Key": f"uploads/user_{user_id}/{filename}",
        "ContentType": "image/jpeg",
    },
    ExpiresIn=300,  # 5 minutes
)
# Client POSTs directly to S3 — your server never handles the file bytes

# Download with streaming (don't load entire file into memory)
response = s3.get_object(Bucket="my-bucket", Key="large-file.csv")
for chunk in response["Body"].iter_chunks(chunk_size=1024*1024):
    process_chunk(chunk)
```

## Lambda: Serverless Functions (11.8-11.9)

```python
# Lambda function (Python)
import json
import boto3

def handler(event, context):
    """
    Lambda handler.
    event: the triggering event (API Gateway request, S3 event, SQS message, etc.)
    context: Lambda runtime info (remaining time, function name, etc.)
    """
    # API Gateway event
    if "httpMethod" in event:
        path = event["path"]
        method = event["httpMethod"]
        body = json.loads(event.get("body") or "{}")

        # Process request
        result = process(body)

        return {
            "statusCode": 200,
            "headers": {"Content-Type": "application/json"},
            "body": json.dumps(result),
        }

    # SQS event
    if "Records" in event:
        for record in event["Records"]:
            message = json.loads(record["body"])
            process_message(message)
        return {"batchItemFailures": []}  # return failed message IDs

    # S3 event
    if event.get("source") == "aws.s3":
        for record in event["Records"]:
            bucket = record["s3"]["bucket"]["name"]
            key = record["s3"]["object"]["key"]
            process_s3_object(bucket, key)
```

**Lambda cold starts:**

```
COLD START TIMELINE:
  Container startup: ~50-300ms (Lambda service)
  Runtime init (Python): ~100-500ms (loading interpreter)
  Function init (your code): import time + connection setup

Total cold start: 100ms (Go/Rust) to 3-10s (Java JVM)

COLD START MITIGATION:
  1. Use lighter languages (Python/Node < Java)
  2. Provisioned concurrency: pre-warm N Lambda instances
  3. Keep Lambda warm: ping every 5 min (not ideal)
  4. GraalVM native for Java: 50ms cold start
  5. Use containers on ECS Fargate for long-running workloads
```

## SQS + SNS: Decoupled Architecture (11.10-11.11)

```python
import boto3

sqs = boto3.client("sqs")
sns = boto3.client("sns")

# SQS: queue for async processing
def enqueue_order(order_id: int):
    sqs.send_message(
        QueueUrl="https://sqs.us-east-1.amazonaws.com/123/orders",
        MessageBody=json.dumps({"order_id": order_id}),
        MessageGroupId="orders",      # FIFO queue: ordering within group
        MessageDeduplicationId=str(order_id),  # FIFO: prevent duplicates
    )

# SQS consumer (Lambda or ECS task)
def process_queue():
    response = sqs.receive_message(
        QueueUrl="...", MaxNumberOfMessages=10, WaitTimeSeconds=20  # long polling
    )
    for msg in response.get("Messages", []):
        try:
            order_id = json.loads(msg["Body"])["order_id"]
            fulfill_order(order_id)
            sqs.delete_message(QueueUrl="...", ReceiptHandle=msg["ReceiptHandle"])
        except Exception as e:
            # Don't delete message — it returns to queue after visibility timeout
            logger.error("Failed to process order", error=str(e))

# SNS: fan-out to multiple subscribers
def publish_event(event_type: str, data: dict):
    sns.publish(
        TopicArn="arn:aws:sns:us-east-1:123:events",
        Message=json.dumps({"type": event_type, "data": data}),
        MessageAttributes={
            "event_type": {"DataType": "String", "StringValue": event_type}
        },
    )
# SNS → fan-out to: SQS (for workers), Lambda (for notifications), email, SMS
```

## Cloudflare Workers: Edge Computing (11.35)

```javascript
// Deploy JavaScript at 300+ global PoPs
// Cold start: <1ms (V8 isolates, not containers)

export default {
    async fetch(request, env) {
        const url = new URL(request.url);

        // Run near the user — geo-routing
        if (url.pathname.startsWith("/api/")) {
            // Auth at edge (validate JWT without hitting origin)
            const token = request.headers.get("Authorization")?.slice(7);
            if (!token || !await validateJWT(token, env.JWT_SECRET)) {
                return new Response("Unauthorized", { status: 401 });
            }

            // Forward to origin with user info
            const response = await fetch(env.ORIGIN_URL + url.pathname, {
                method: request.method,
                headers: {
                    ...request.headers,
                    "X-User-Id": getUserIdFromJWT(token),
                },
                body: request.body,
            });
            return response;
        }

        // Serve static assets from edge cache
        return fetch(request);
    }
}
```

**Cloudflare's edge data primitives:**
- **KV:** globally distributed key-value store (~50ms eventual consistency)
- **D1:** SQLite databases at the edge (SQL queries, serverless)
- **R2:** S3-compatible object storage with no egress fees
- **Durable Objects:** stateful objects with strongly consistent storage

## Cost Optimization (11.40)

```
AWS COST REDUCTION STRATEGIES:

EC2/FARGATE:
  - Use Spot Instances (70-90% cheaper, can be interrupted)
    → Good for: batch jobs, non-critical workers, dev environments
  - Reserved Instances: 1-3 year commitment, 30-70% cheaper
    → Good for: stable production workloads
  - Graviton (ARM) instances: 20% cheaper, often faster
    → Most workloads run on ARM without changes

S3:
  - Intelligent-Tiering: auto-moves infrequent data to cheaper storage
  - S3 Glacier for archives: $0.004/GB/month vs $0.023/GB for standard
  - Transfer Acceleration only when needed (expensive)

DATA TRANSFER:
  - Same AZ: free. Cross-AZ: $0.01/GB. Cross-region: $0.02/GB.
  - Use VPC endpoints for S3/DynamoDB (avoid NAT Gateway charges)
  - Compress API responses (gzip reduces bandwidth costs)

LAMBDA:
  - Right-size memory (higher memory = more CPU = faster = cheaper per request)
  - x86 vs ARM64: ARM is 20% cheaper, often faster

MONITORING COSTS:
  - Set CloudWatch billing alarms
  - Enable AWS Cost Explorer
  - Use AWS Compute Optimizer for right-sizing recommendations
  - Cost Anomaly Detection: alert on unexpected spend
```
